# AI 内容理解偏差："修改完善"被理解为"填空"

## 基本信息

| 项目 | 内容 |
|------|------|
| **日期** | 2026-04-04（吴律反馈） |
| **严重程度** | 中（影响文书质量） |
| **影响范围** | 所有文书生成类 Skill |
| **状态** | ✅ 已解决 |
| **反馈来源** | 吴律 |

## 问题描述

吴律反馈：让 AI "修改完善"一份答辩状时，AI 把"修改完善"理解为"把空白处填上内容"，而不是"对已有内容进行法律层面的优化和改进"。

期望行为：
- 理解已有答辩状的内容和逻辑
- 针对法律层面提出改进建议（法条引用、论证逻辑、事实补充等）
- 输出优化后的版本

实际行为：
- 把空白处（如当事人信息、日期等占位符）直接填上内容
- 不分析已有内容的法律问题
- 缺少实质性的法律优化

## 根因分析

1. **Skill 指令不够明确**：对"修改"和"填空"的区分不够清晰
2. **AI 取最短路径**：AI 倾向于选择最简单的执行路径（填空比法律分析更容易）
3. **缺少强制步骤**：没有"先理解已有内容，再提出修改建议"的强制步骤
4. **术语歧义**："修改完善"在 AI 的训练数据中可能更常见于"填写模板"的场景

## 解决方案

1. **添加强制理解步骤**：Skill 中添加"先输出当前内容理解摘要，再提出修改建议"的强制步骤
2. **区分两种场景**：从零生成 vs 修改已有文书，用不同的指令前缀明确区分
3. **多模型交叉验证**：引入审查 Agent，确保理解一致
4. **明确术语定义**：在 Skill 中定义"修改完善"= 对已有内容进行法律层面的优化，不是填空

## 经验教训

- AI 的"理解"是概率性的，需要用结构化约束来锁定
- 关键术语（修改、完善、优化）需要明确定义，不能依赖 AI 的"常识"
- 多模型交叉验证是防止单一模型理解偏差的有效手段
- 用户反馈中"AI 不理解我的意思"往往是指令不够具体，不是 AI 能力问题

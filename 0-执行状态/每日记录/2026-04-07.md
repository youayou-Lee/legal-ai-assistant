# 2026-04-07 工作记录

## 吴律反馈（杨刚案）

吴律用杨刚案真实材料测试，反馈三个问题：

### 问题1：上诉状结构+证据引用差距

- AI版（GLM-5默认生成）vs 律师修订版有显著差距
- **论证顺序**：AI把"被上诉人过错"放在鉴定违法前面，律师把鉴定违法放第二位（优先级不同）
- **编号体系**：AI版混用（一→(一)→无编号），律师版统一（一→1.→首先/其次）
- **证据引用**：AI用模糊的"证据5"，律师用具体描述+论证结合
- **论证深度**：AI罗列所有可能点，律师更精炼聚焦
- 结论：不是生成不出来，是需要专门Skill约束结构、论证优先级、证据引用格式
- 参考材料：`/home/you/文档/Obsidian Vault/反馈材料/杨刚材料包/一审材料/` 下两个上诉状

### 问题2：法条错误（GLM-5系统性问题）

- AI版引用民诉法76条关于鉴定程序——**实际76条是"证人出庭"**
- 正确应该是79条（申请鉴定与专门性问题）
- 智谱清言开联网搜索、Qwen、DeepSeek都能正确回答
- **根因**：GLM-5默认Agent不联网搜索时，法条内容靠训练数据"猜"，有幻觉
- **影响范围**：所有Skill（起诉状、答辩状、预判等）引用法条时都有这个风险
- 法条库MD方案只覆盖婚姻财产类，其他领域未覆盖

### 问题3：MD→DOCX格式问题

- 默认输出MD对律师不友好，律师要的是docx
- MD转docx时出现多余横线（`---` 变成水平线）、排版混乱
- 所有Skill的输出都有这个问题

## 诊断：系统性问题

三个问题都是跨Skill的系统性问题，不只影响上诉状：
1. 法条幻觉 → 所有文书生成Skill
2. 结构约束不足 → 所有文书生成Skill
3. 输出格式 → 所有Skill

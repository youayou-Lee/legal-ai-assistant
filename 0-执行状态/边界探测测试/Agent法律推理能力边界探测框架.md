# 通用Agent法律推理能力边界探测框架

> 从本项目实际测试中提炼的可复用方法论。
> 核心教训来源：继承案假阳性（阶段九）、过度泛化（阶段十）、法条库A/B验证。

---

## 一、能力分级模型

Agent在法律领域的推理能力不是"行/不行"的二值判断，而是一个四层金字塔。每一层有独立的失败模式，需要不同的测试方法和补齐策略。

```
        ┌──────────┐
        │ L4 复杂   │  多层交叉、跨时间线
        ├──────────┤
        │ L3 精算   │  定量计算、公式应用
        ├──────────┤
        │ L2 推理   │  多步骤分析、顺序纪律
        ├──────────┤
        │ L1 概念   │  精确法律概念和分类
        ├──────────┤
        │ L0 常识   │  方向性判断、大方向对
        └──────────┘
```

### L0：常识判断（Common Sense）

**定义**：基于一般社会常识和法律直觉，做出方向性正确的判断。

**表现特征**：
- "婚前财产归个人" ✅
- "明知有危险不处理，有过错" ✅
- "买卖不破租赁" ✅
- "用于家庭生活的债务应该一起承担" ✅

**典型通过率**：90%+（几乎所有现代Agent都能通过）

**测试意义**：低价值。通过L0不说明任何问题，只是确认基线。

---

### L1：概念精度（Legal Concepts）

**定义**：能使用精确的法律概念和分类框架，而非近似的日常用语。

**表现特征**：
- "通谋虚伪表示"vs"以合法形式掩盖非法目的"（新法vs旧法概念）
- "新债清偿"vs"代物清偿"（两个相似但有区别的概念）
- "投资收益"vs"自然增值"（知道概念但不确定判断标准）

**失败模式**：方向对但概念模糊
- 知道"规避担保程序不对"，但说不清这叫"通谋虚伪表示"（民法典146条）
- 知道"婚后赚的钱可能要分"，但分不清"孳息""自然增值""投资收益"

**已验证数据**（本项目）：
- 合同纠纷：概念近似但不精确（4次）
- 婚姻财产：标注不确定但方向正确（5次）

---

### L2：推理纪律（Reasoning Discipline）

**定义**：能按正确顺序执行多步分析，不跳过必要的前置步骤。

**表现特征**：
- 继承案：先铺完整死亡→继承时间线，再算份额
- 合同案：先确认表面形式vs真实意思，再判断效力
- 劳动案：先确认法律关系性质，再算赔偿

**失败模式**：
1. **静态思维**：把"份额"当固定标签，不追问动态累积
2. **跳过时间线**：不先铺时间线就直接跳到计算
3. **锚定效应**：被最显性信息锁定，停止追问

**关键区分**：
- L2失败 = **知道规则但不系统应用**（缺纪律）
- L1失败 = **不知道规则**（缺知识）
- 这两种失败需要完全不同的解决方案

**已验证数据**：
- 继承案：简单题过（L1通过），复杂题错（L2失败）→ 假阳性
- 合同纠纷：70%纪律缺失 + 30%知识缺失
- 劳动争议：推理纪律缺失为主

---

### L3：精确计算（Quantitative Precision）

**定义**：能执行法律领域的定量计算——公式正确、参数准确、结果精确。

**表现特征**：
- 劳动赔偿：2N经济赔偿金（违法解除）的计算
- 加班费：平时150%、周末200%、法定节假日300%
- 继承份额：多层继承的精确分数计算
- 利息：清偿抵充顺序下的分段利息计算

**失败模式**：
- 定性对但定量缺（"知道违法解除但不知道2N"）
- 知道公式但不调用（清偿抵充顺序：先利息后本金）

**已验证数据**：
- 劳动争议：定性对但缺计算标准
- 起诉状v1：简单扣除而非清偿抵充计算

---

### L4：复杂场景整合（Complex Integration）

**定义**：能处理多层法律关系交叉、跨时间线、多主体的综合案件。

**表现特征**：
- 三代多层继承（陈继权案）：遗产范围确认→代位继承→份额计算
- 混合财产分割：个人财产+共同财产+增值+孳息+投资收益
- 公司担保纠纷：越权代表+善意相对人+合同效力+赔偿责任

**失败模式**：
- 只看到最表层法律关系，不追问隐藏层
- 处理单一关系时正确，多关系交叉时遗漏

**这是律师日常面对的真实复杂度。** L0-L3测试通过不代表L4能通过。

## 二、测试方法：三层漏斗

```
第一层：快速摸底（广度）
  ↓ 筛出"不通过"和"部分通过"的类型
第二层：详细诊断（深度）
  ↓ 区分知识缺失 vs 纪律缺失
第三层：A/B验证（干预效果）
  ↓ 量化干预手段的效果
```

### 第一层：快速摸底

**目的**：用最少成本识别哪些案件类型需要深入测试。

**方法**：
- 每个案件类型 **1道中等难度题**
- 重点：**不要用简单题**（会产生假阳性，继承案的教训）
- 评分维度4项：步骤完整性 | 步骤顺序 | 结论正确性 | 概念准确性

**评分标准**：
| 评定 | 含义 | 下一步 |
|------|------|--------|
| ✅ 通过 | 4项全部通过 | 进入L4验证（用真实复杂案例） |
| ⚠️ 部分通过 | 有1-2项不通过 | 进入第二层详细诊断 |
| ❌ 不通过 | 3项以上不通过 | 直接进入第二层 |

**⚠️ 关键规则**：
- **"✅通过"必须用真实复杂案例验证**，不能仅凭标准化题目判定
- 简单题通过 ≠ 该类型Agent能力过关（继承案教训）
- 题目设计要覆盖L1-L3至少两个层级

---

### 第二层：详细诊断

**目的**：区分"知识缺失"和"纪律缺失"，确定干预方向。

**方法**：
1. **设计复杂案例**：每个类型1道，比第一层更复杂（多主体/多时间线/多层关系）
2. **写专家标准答案**：必须由法律专业人士或详细法条研究确认
3. **模拟Agent分析**：不给任何法律专项prompt，记录Agent自然回答
4. **追问探测**：针对Agent标注"不确定"或出错的地方，直接追问相关法律概念
5. **诊断归因**：

```
追问后Agent立刻修正？
├── 是 → 纪律缺失（知道但不用）
│   └── 解决方向：写强制步骤模板（轻投入）
└── 否 → 知识缺失（确实不知道）
    └── 解决方向：注入法条库/知识（中投入）
```

**追问探测的关键**：
- 不是问"你能再想想吗"（这只是在催促）
- 而是直接点出法律概念："你知道XX条司法解释吗？"（探测知识边界）
- 如果Agent秒答正确 → 纪律问题
- 如果Agent仍然答不上 → 知识问题

**已验证的诊断结果**（本项目）：

| 类型 | 归因 | 依据 |
|------|------|------|
| 合同纠纷 | 70%纪律 + 30%知识 | 知道"规避担保"但不用"通谋虚伪表示"；追问后能补充但不够精确 |
| 劳动争议 | 纪律缺失为主 | 知道概念但缺计算标准，追问后能识别公式 |
| 婚姻财产 | **70%知识 + 30%纪律** | 追问后仍不知道司法解释29条、78条——真正的知识盲区 |
| 继承纠纷 | 纪律缺失 | 知道代位继承规则但不先铺时间线 |

---

### 第三层：A/B验证

**目的**：量化干预手段的实际效果。

**方法**：
- **A组**（对照组）：无任何法律专项prompt
- **B组**（实验组）：注入干预手段（法条库/步骤模板/Skill）
- 对比指标：
  - "不确定"标注数量
  - 法条引用覆盖度
  - 结论正确率
  - 计算精确度

**已验证数据**（婚姻财产法条库）：

| 指标 | A组（无法条库） | B组（有法条库） | 改善 |
|------|---------------|---------------|------|
| "不确定"标注 | 8处 | 1处 | **-87.5%** |
| 法条引用 | 0 | 每项1-2处 | 全覆盖 |
| 增值性质区分 | 完全模糊 | 精确引用78条 | 质变 |
| 父母出资定性 | 不敢断言 | 引用29条给判断流程 | 质变 |

**A/B验证的设计原则**：
- 同一个案例，同一个Agent，只变量是"有无干预"
- 指标必须量化（不能只说"感觉好多了"）
- 至少跑1个案例验证，2-3个更可靠

## 三、案例选择标准

### 核心原则：简单题产生假阳性

继承案的教训：标准化测试题（一代继承、关系清晰）Agent轻松通过 → 判定"✅通过"。但用真实案例（三代多层、时间线交叉）Agent第一次就错。

**结论**：测试案例的复杂度必须≥律师日常遇到的平均水平，不能低于这个基线。

### 案例来源（按推荐优先级排列）

| 优先级 | 来源 | 优点 | 缺点 |
|-------|------|------|------|
| **1** | **律师真实案例（脱敏）** | 复杂度真实、通过=实战可用 | 获取成本高、需要脱敏 |
| **2** | **最高院指导性案例** | 权威、有裁判要旨 | 可能偏简单（经过简化） |
| **3** | **法考真题（主观题）** | 难度标定、覆盖面广 | 可能比实务简单 |
| **4** | **AI生成案例** | 快速、可定制难度 | 可能有逻辑漏洞，需专家审核 |

### 案例复杂度分级

**每个案件类型的测试案例需要两个版本**：

| 版本 | 复杂度 | 用途 |
|------|--------|------|
| **标准版** | 中等（1-2层法律关系、1-2个时间节点） | 快速摸底 |
| **复杂版** | 高（3+层法律关系、多时间节点、多主体） | 详细诊断 + 避免假阳性 |

### 各类型复杂版测试要点

| 案件类型 | 复杂版必须包含的元素 |
|---------|-------------------|
| 继承纠纷 | 3代以上、代位继承、遗嘱+法定混合、债务清偿 |
| 合同纠纷 | 通谋虚伪表示、越权代表、多份合同关联 |
| 婚姻财产 | 父母出资、婚前财产婚后增值、混合财产剥离、共同债务认定 |
| 劳动争议 | 违法解除赔偿计算、加班费多倍率、竞业限制 |
| 侵权纠纷 | 多因一果、纯粹经济损失、过错推定 |
| 物权纠纷 | 抵押+租赁+买卖交叉、善意取得、优先购买权 |

## 四、验证方法

### 四维评分体系

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| **步骤完整性** | 25% | 有无遗漏必要分析步骤 |
| **步骤顺序** | 25% | 步骤执行顺序是否正确（先A后B） |
| **结论正确性** | 25% | 最终结论与标准答案是否一致 |
| **概念准确性** | 25% | 法律概念是否精确（非近似） |

### 评分流程

```
1. 准备标准答案（专家/法条研究确认）
2. Agent在无专项prompt下分析
3. 逐维度对比评分（✅/⚠️/❌）
4. 计算综合评定
5. 如有❌或⚠️，执行追问探测
6. 归因：知识缺失 or 纪律缺失 or 混合
```

### 追问探测协议

追问不是简单的"你能再想想吗"，而是**精准探测知识边界**：

| 追问类型 | 示例 | 探测目标 |
|---------|------|---------|
| **概念追问** | "你知道'通谋虚伪表示'这个概念吗？" | L1概念精度 |
| **法条追问** | "婚姻家庭编司法解释第29条你了解吗？" | 知识vs纪律 |
| **步骤追问** | "你是不是应该先确认遗产范围再算份额？" | L2推理纪律 |
| **计算追问** | "清偿抵充的顺序是什么？" | L3精确计算 |

**追问结果的归因规则**：
- 追问后秒答正确 → **纪律缺失**（知道但不用）→ 方案：步骤模板
- 追问后回答改善但仍有错 → **混合型** → 方案：步骤模板 + 法条库
- 追问后仍答不上 → **知识缺失**（不知道）→ 方案：法条库注入

### 假阳性防范清单

以下情况必须警惕假阳性：

- [ ] 只用标准化题目，没用真实复杂案例
- [ ] 只测了1道题就下结论
- [ ] 题目只有1层法律关系
- [ ] 题目事实清晰、无争议空间
- [ ] 从单一案件类型推广到所有类型
- [ ] 追问和初始回答用了同一个对话（上下文污染）

## 五、干预策略选择

根据诊断结果选择干预策略：

```
诊断结果
├── 纪律缺失为主（>70%）
│   └── 方案：强制步骤模板（Skill Prompt中的强制推理步骤）
│       ├── 投入：低（1-2天）
│       └── 效果：解决步骤跳过、顺序错误
│
├── 知识缺失为主（>70%）
│   └── 方案：法条库MD + prompt调用机制
│       ├── 投入：中（2-3天）
│       └── 效果：消除概念模糊、填补知识盲区
│
└── 混合型
    └── 方案：步骤模板 + 法条库（两者结合）
        ├── 投入：中（3-5天）
        └── 效果：同时解决纪律和知识问题
```

### 干预效果的验证标准

干预后必须重新跑A/B测试：

- **纪律型**：步骤完整性从❌/⚠️→✅，步骤顺序从跳步→有序
- **知识型**：法条引用从0→全覆盖，"不确定"标注减少>80%
- **混合型**：两个维度同时改善

## 六、测试执行清单

### 准备阶段

- [ ] 确定要测试的案件类型清单
- [ ] 为每个类型准备标准版+复杂版测试案例
- [ ] 准备专家标准答案（法条研究+律师确认）
- [ ] 准备追问探测问题清单

### 执行阶段

- [ ] 第一层：快速摸底（每类型1题，中等难度）
- [ ] 标记"✅通过"的类型 → 用复杂版验证（避免假阳性）
- [ ] 标记"⚠️部分通过"和"❌不通过" → 进入第二层
- [ ] 第二层：详细诊断（复杂案例 + 追问探测）
- [ ] 归因每个类型的失败模式

### 干预与验证

- [ ] 根据归因选择干预策略（步骤模板/法条库/混合）
- [ ] 实施干预
- [ ] 第三层：A/B验证（同一案例，有/无干预对比）
- [ ] 量化效果（不确定标注、法条引用、结论正确率）

## 附录：关键教训索引

1. **假阳性陷阱**：简单测试题会让Agent看起来比实际更强。必须用复杂版验证。→ 继承案教训
2. **过度泛化陷阱**：从一个类型的诊断推广到所有类型。每类必须独立诊断。→ 阶段十教训
3. **归因决定方向**：归因为"知识缺"→建知识库（重投入）；归因为"纪律缺"→写流程约束（轻投入）。诊断错误比问题本身更危险。→ 阶段九教训
4. **追问探测是关键**：不是问"能再想想吗"，而是直接点出法律概念看Agent反应。秒答=纪律问题，答不上=知识问题。→ 详细诊断方法论
5. **A/B验证不可跳过**："感觉好多了"不等于真的好。必须量化对比。→ 法条库验证
6. **输出不好改Harness不改输出**：这是Harness Engineering的核心原则。→ 阶段八认知

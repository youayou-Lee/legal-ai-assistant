# 通用Agent法律专业知识边界探测

> 目的：搞清楚通用Agent在哪些法律任务类型上因为缺乏专业知识而做错，针对性补齐。

---

## 背景认知（2026-04-04 讨论）

**核心区分：推理能力 ≠ 专业知识**

- LLM的通用推理能力已经很强（世界观、价值观、阅历带来的推理）
- LLM缺的是**法律领域专业知识**——"做继承案件必须先确认遗产范围再算份额"这类行业知识
- 当LLM不知道专业知识时，会凭通用理解给一个"看起来合理但专业上不对"的答案
- 像"从材料中提取人物关系"、"总结争议焦点"这类任务不需要专业知识，通用能力就能做，不属于本次测试范围

**已有失败案例**：
- 黄律继承析产：AI跳过"先确认遗产继承法律关系"直接算份额
- 吴律婚内财产协议：AI把"修改完善"理解成"填空"（结构性调整缺失）

---

## 测试范围

### 测的是什么

- 通用Agent是否掌握特定案件类型所需的法律专业知识
- 在需要专业知识的推理任务中，推理路径是否正确

### 不测什么

- ~~通用推理能力~~（LLM已经很强）
- ~~法条引用准确度~~（LLM幻觉问题，解决方案是RAG+复审）
- ~~指令理解能力~~（短期不需要，用户会表达清楚）
- ~~事实梳理/信息提取~~（不需要专业知识，通用能力可做）

---

## 分类维度

**按案件类型分类**（每个案件类型有不同的专业知识体系）：

| 案件类型 | 需要的专业知识示例 | 测试重点 |
|---------|------------------|---------|
| 继承纠纷 | 遗产范围确认→继承顺序→份额计算，每步不能跳 | 推理步骤顺序 |
| 合同纠纷 | 合同效力判断→违约认定→损失计算 | 法律关系认定 |
| 婚姻财产 | 共同财产vs个人财产认定→分割原则→特殊情况处理 | 概念区分 |
| 侵权纠纷 | 过错认定→因果关系→损害赔偿计算 | 逻辑链完整性 |
| 劳动争议 | 劳动关系认定→权利义务→赔偿标准 | 法定程序意识 |
| 物权纠纷 | 权属认定→优先顺序→保护方式 | 权利冲突处理 |

## 测试方法

### 两步走

#### 第一步：快速摸底

- 每个案件类型选 **1道法考真题**（案例分析题，有标准答案）
- 用通用Agent（不加任何法律专项prompt）跑一遍
- 对照标准答案，判断通过/不通过
- **通过标准**：推理步骤和结论与标准答案一致
- **产出**：一张表

| 案件类型 | 题目来源 | AI输出 | 标准答案 | 通过？ | 错误简述 |
|---------|---------|--------|---------|-------|---------|
| 继承纠纷 | 法考20xx年 | ... | ... | ❌ | 跳过遗产范围确认 |
| 合同纠纷 | 法考20xx年 | ... | ... | | |
| ... | | | | | |

#### 第二步：详细诊断

- 对摸底中 ❌ 的案件类型，每个类型再用 **3-5道题** 深入测试
- 记录每道题的 **具体错误模式**：

```
错误模式分类：
- 跳步：缺少必要的前置步骤
- 顺序错：步骤有但顺序不对
- 概念错：对法律概念理解有误
- 完全不知道：没有相关知识，凭通用理解瞎猜
- 部分正确：大方向对但细节有误
```

- **产出**：每个失败案件类型的**能力缺口报告**

```
能力缺口报告模板：

案件类型：继承纠纷
测试题数：5道
通过：1道 / 不通过：4道

错误模式分布：
- 跳步：3次（跳过遗产范围确认直接算份额）
- 概念错：1次（混淆代位继承和转继承）
- 部分正确：1次（步骤对但份额计算公式错）

缺失的知识点：
1. 继承案件必须按"遗产范围→继承顺序→份额计算"三步走
2. 夫妻共同财产需先析产再继承
3. 代位继承的适用条件

补齐建议：
- [方案A] prompt增强：在分析类任务中加入强制步骤模板
- [方案B] 专项Skill：继承纠纷分析Skill（含完整推理框架）
- [方案C] 知识库注入：继承法核心知识条目
```

## 测试题目来源

| 来源 | 优势 | 劣势 |
|------|------|------|
| **法考（司法考试）历年真题** | 标准答案权威、覆盖面广、难度梯度清晰 | 题目偏学术，和实务有差距 |
| 裁判文书网公开案例 | 真实案例，实务导向 | 没有标准答案，需要律师判断 |
| 已有黄律/吴律案件 | 有律师反馈，知道问题在哪 | 数量少，覆盖面窄 |

**执行顺序**：法考真题摸底 → 公开案例补充 → 已有案件验证

---

## 验证方式

- **你能做的**：对照标准答案看AI输出的**结构和步骤是否一致**，不需要懂法律内容
- **拿不准的**：攒一批一起发给吴律/黄律看
- **对比维度**：
  1. 步骤是否完整（有没有跳步）
  2. 步骤顺序是否正确
  3. 最终结论是否正确
  4. 推理过程中的概念是否准确

---

## 后续应用

诊断出能力缺口后，按"缺什么补什么"：

| 缺口类型 | 补齐策略 | 示例 |
|---------|---------|------|
| 零星知识点 | prompt约束 | "继承案件必须先确认遗产范围" |
| 系统性知识缺失 | 专项Skill | 继承分析Skill（含完整推理框架） |
| 知识面太广 | 法律知识库 | RAG + 专业数据库 |

## 执行计划

1. 🧑 筛选法考真题（每个案件类型1道，6个类型共6道）
2. 🤖 设计测试框架、评分标准、输出模板
3. 🤖 用通用Agent跑全部测试题
4. 🧑 对照标准答案判断通过/不通过
5. 🤖 对失败类型做详细诊断（每个3-5道题）
6. 🤖 整理能力缺口报告
7. 🧑 确认报告，决定补齐策略

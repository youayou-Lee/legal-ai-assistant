## 一、案件代理全流程：律师（我会做） vs. AI（帮我做）

### (一) 民事案件

#### 一审原告

| 律师（我会做） | AI（帮我做） |
|--------------|------------|
| 1. 收集材料、梳理事实、确定法律关系 | 1. 输入案情后复现律师工作，供查漏补缺 |
| 2. 确定当事人、管辖、时效、诉请、证据链完整性 | 2. 生成起诉状、证据目录及证明目的 |
| 3. 风险评估及提示 | 3. 匹配法条、司法解释、类案裁判要点 |
| | 4. 预判被告答辩点及抗辩证据 |
| | 5. 总结争议焦点及法官发问清单 |
| | 6. 生成庭审提纲 |

#### 一审被告

| 律师（我会做） | AI（帮我做） |
|--------------|------------|
| 1. 核对事实及证据真实性 | 1. 分析材料生成答辩状 |
| 2. 分析原告诉请是否成立、证据是否有瑕疵 | 2. 总结争议焦点 |
| 3. 审核主体资格、管辖、时效、法律关系、过错、因果关系、数额等 | 3. 指出原告诉请漏洞、证据缺陷及质证意见 |
| | 4. 生成被告证据目录（含反驳证据） |
| | 5. 匹配抗辩法条、类案观点 |
| | 6. 预判原告质证意见 |
| | 7. 生成庭审提纲 |

#### 二审上诉人

| 律师（我会做） | AI（帮我做） |
|--------------|------------|
| 1. 收集一审材料（判决、诉状、证据等），申请复制笔录 | 1. 生成上诉状 |
| 2. 与委托人确认事实 | 2. 提炼一审错误（事实/法律/程序） |
| 3. 审查事实认定、法律适用、程序是否有误；确定上诉点及新证据 | 3. 生成上诉证据目录参考 |
| | 4. 匹配二审改判法条及案例 |
| | 5. 预判被上诉人答辩观点 |
| | 6. 生成二审庭审及法官询问提纲 |

#### 二审被上诉人

| 律师（我会做） | AI（帮我做） |
|--------------|------------|
| 1. 同上 | 1. 生成二审答辩状 |
| 2. 审查上诉理由是否成立 | 2. 逐条拆解上诉理由不成立依据 |
| 3. 巩固一审判决正确性 | 3. 匹配维持原判相关法条 |
| 4. 反驳上诉逻辑 | 4. 预测上诉人新证据及辩论重点 |
| | 5. 生成二审庭审提纲及法官询问清单 |

### (二) 刑事案件

#### 侦查阶段

| 律师（我会做） | AI（帮我做） |

| 律师（我会做） | AI（帮我做） |

## 二、AI产出的必要性评价

### 有必要（核心价值）

- 案情归纳及分析
- 文书及证据目录生成
- 风险分析、诉讼策略分析
- 法条匹配、类案匹配、量刑参考
- 庭审提纲（含争议焦点、发问清单）
- 合同审查、个性化建议（律师偏好）
- 纠错功能

### 无必要（干扰信息）

- 过多展示过程性的东西
- 简单问题复杂化，堆砌无必要的文字
- 胜诉率/败诉率预测

## 三、效果优化需求（待改进点）

### 能力层面
- 提升事实梳理能力及法律适用准确性
- 加强案件分析及文书内容的深度

### 语言层面
- 生成内容需更符合"法言法语"
- 避免凭空"造法"或法条更新不及时导致错误

### 功能层面
- 降低对指令细化程度的过高要求
- 实现单个文件拖拽分析
- 提升响应速度
- 增强文件格式包容度
- 改进脱敏效果
- 提高类案匹配精准度

### 稳定性层面
- 解决文件过多、过大时无法正常工作的问题

---

## 四、对助理（AI）的审核标准

律师希望只需审核以下核心内容：

1. 当事人是否适格或是否遗漏当事人
2. 是否遗漏诉讼请求、答辩要点是否全面
3. 事实理由是否完整、准确
4. 法律适用是否准确
5. 是否已形成完整证据链、证明目的是否清晰

## 关键分析（待与黄律进一步讨论）

### 这份文档揭示了什么

1. **律师对AI的期望非常具体** — 不是"帮我分析案件"这种模糊需求，而是每个阶段都有明确的具体任务
2. **"预判"是高频需求** — 预判答辩点、预判质证意见、预判辩论重点。律师最怕的是"没想到对方会出这招"
3. **律师的核心审核点只有5个** — 这5个点就是AI输出的质量底线
4. **律师明确说"不需要过程"** — 这跟你之前做OpenClaw时"过程不可见导致律师失去耐心"的观察是一致的。律师不要看到过程，但要看到可靠的结果
5. **刑事和民事的需求结构完全不同** — 民事是阶段性的（一审/二审），刑事是流程性的（侦查/起诉/审判），Skill设计需要区分

## 深度讨论：生产层替代后律师的认知怎么办

### 问题起源

愿景里的两层模型——生产层（AI做）和责任层（律师做）——有一个隐藏假设被打破了：**律师责任层的能力有一部分是从生产层工作中生长出来的。**

律师翻阅200页卷宗不只是"处理材料"，同时也是在建立案件心智模型——直觉、掌控感、"这个案子哪里不对劲"的判断力。如果AI全包生产层，律师拿到的是精美报告，但失去的是**认知过程**。

Review不够的原因：
- **被动vs主动**：review是验证别人的逻辑，不是主动建构自己的心智模型
- **Unknown unknowns**：AI没展示的，律师永远看不到，失去了"偶遇关键线索"的机会
- **庭审准备**：做过生产层工作的律师能在庭上即时响应，仅靠review的律师是"看过总结"的人

### 新认知：可能需要三层模型

原来的两层漏了一层：

```
生产层（AI）→ 认知层（AI+律师协作）→ 责任层（律师）
```

中间的**认知层**不是被动步骤，是AI和律师持续交互的空间。AI的角色从"生产者"变成"策展人+引导者"——发现线索但不给结论，设计律师思考的路径而不是替代律师思考。

### 律师现实：他们自己拆任务让AI做

和律师沟通后发现，他们的本能是**自己把控任务拆解**，把子任务扔给AI。这和愿景中"AI包揽生产层"的假设天然冲突。

律师的原话："就算AI生成的100%正确，我也要复查。"——**正确性不是问题，掌控感才是。这是行业特点，不分案件复杂度。**

### 两个方向都认可（待融入产品）

#### 方向A：AI做策展人+引导者（交互范式）

核心转变：AI从"替你想"变成"陪你看"。

当前模型的问题是AI做了一碗饭端上来，律师只能尝一口说行或不行。律师没有参与烹饪过程，对这道菜的理解停留在"味道好不好"，而不是"怎么做的、为什么这样搭配"。

方向A的改法：AI负责备料、切菜，关键环节让律师亲自操刀。具体来说：

**AI做的（不变）：**
- OCR、文本提取、文档结构化
- 把200页卷宗变成可检索、可导航的结构
- 发现潜在关联、标记异常、整理原始数据

**改变的是AI的输出方式——从"给结论"变成"引思考"：**

例子1——证人矛盾：
```
// 现在的模式（生产者——AI给结论）
"证人A的两次笔录存在矛盾：第一次说他当晚在家，第二次说在酒吧。"

// 方向A的模式（策展人——AI引关注，律师自己发现）
"证人A有两次笔录，间隔7天。我标注了几个值得关注的时间节点。
你先看第一份pp.12-15，再看第二份pp.45-47。"
```

AI发现了线索但没有下结论。律师看到两份笔录后自己发现矛盾——这个"发现"的瞬间就是认知构建的过程。结论一样，但认知所有权不同。

例子2——证据链断裂：
```
// 现在的模式
"本案证据链存在断裂：指控X的动机缺乏直接证据支撑。"

// 方向A的模式
"关于被告动机，卷宗里有三种说法：
- 侦查机关的认定（起诉书p.3）
- 被告本人的辩解（笔录p.22）
- 被害人家属的描述（笔录p.56）
这三个版本差异很大。我帮你整理了对比表，你看一下。"
```

律师看到对比表，自己得出"证据链断裂"的判断。他不是在审查别人的判断，而是在做出自己的判断。

例子3——银行流水分析（小任务场景）：
```
// 纯任务执行
"1-3月转入总额XX万，转出总额XX万..."

// 方向A偷织入
"1-3月转入总额XX万，转出总额XX万。
另外注意到第3页和第7页有几笔转账时间点比较密集（已标注），
可能和XX有关联，你看一下原始件确认。"
```

律师的反应不是"这个AI改变了我的工作方式"，而是"这个AI还挺敏锐的"。信任就是在这样的交付中慢慢建立的。

**方向A的关键设计原则：**
1. AI只做"发现"和"呈现"，不做"判断"和"结论"
2. AI的输出应该引发律师的思考，而不是替代律师的思考
3. AI说"你看看这里"而不是"这里有问题"
4. 律师是自己"走到"结论的，AI只是铺了路

**和律师当前行为的兼容性：**
- 律师自己拆任务发过来 → 方向A不阻止这个行为
- 但每次AI返回结果时，可以附带一个"我发现你可能还想看的东西"
- 律师可以选择忽略或深入，主导权在他
- 这不需要律师同意任何"范式转变"，转变在好的交付中自然发生

#### 方向B：三层模型（架构认知）

原来的两层模型：

```
生产层（谁做不重要）→ 责任层（必须律师做）
```

这个模型漏了一层。律师从"翻完卷宗"到"出庭辩护"之间，不是一步跳过去的。中间有一个**逐步建立案件心智模型**的过程——翻材料时形成的直觉、发现矛盾时的兴奋、对某个人物的不信任感、对时间线的整体把握。这些东西不是生产层的输出，也不是责任层的动作，它们是律师脑子里的案件认知模型。

修正后的三层模型：

```
生产层（AI）
  ↓ 输出结构化的原始材料
认知层（AI + 律师协作）
  ↓ 输出律师的案件心智模型
责任层（律师）
```

**生产层（AI独占）：**
- 材料处理、OCR、文本提取、文档结构化
- 事实提取、时间线整理、证据编目
- 关联发现、异常标记、数据清洗
- 这层的目标：把杂乱的材料变成可用的结构化数据

**认知层（AI + 律师协作）——这是新增的核心层：**

这层不是一个被动的步骤，而是一个持续交互的空间，可能贯穿整个案件周期。它的特征是：

1. **AI重新包装生产层的原始发现**：不是把原始数据直接倒给律师，而是转化成律师需要主动思考的问题和场景
2. **律师在这个过程中逐步建立对案件的掌控感**：每次交互都在加深他对案件的理解
3. **AI追踪认知覆盖度**：哪些关键节点律师已经处理了，还有哪些没覆盖到
4. **渐进式深化**：从初步全局印象 → 关键矛盾发现 → 细节验证 → 形成完整心智模型

这层可以做的事：
- 案件全局导览：AI整理出"这个案件的几个关键问题"，让律师先建立全局印象
- 关键节点暴露：AI发现异常但不解释，留白让律师去发现意义
- 假设验证引导：律师形成初步判断后，AI提供支持或质疑该判断的材料
- 认知缺口提醒："这个证人的证词你还没看过，他在XX时间点的说法和其他人有出入"

**责任层（律师独占）：**
- 最终判断、签字、出庭、谈判、客户沟通、结果承担
- 这层的质量直接取决于认知层的深度

**方向A和方向B的关系：**
- 方向B是**架构层面的认知**：你的模型漏了一层，必须补上
- 方向A是**交互层面的设计**：这一层具体怎么运作
- A描述的就是B中认知层的具体交互方式
- 两者是同一件事的两个视角，不是二选一

### 落地路径还没决定

**路径一：先改变观念，再改变工具**
- 根因分析：律师自己拆任务的本质原因是需要掌控感
- 只要保证掌控感（认知层设计），就有可能说服律师接受新工作范式
- 类比：20年前码农手搓代码，如今靠AI coding——范式转变需要契机，但会发生

**路径二：先顺着他来，从小任务验证**
- 律师已经告诉你他想要什么——按他的方式先做
- 如果小任务都做不好、律师不满意，谈什么包揽生产层？
- 小任务验证的是**信任和交付质量**，信任积累后才能推动范式转变

**两条路径都合理，需要和律师聊完再决定。**

## 架构反思：OpenClaw的三层架构

> 这篇是之前的思考，今天因为吴律的"没反应"问题重新翻出来的。

### 当前问题

OpenClaw是单Agent阻塞模式：main Agent同时负责对话和执行任务。一旦安排了任务，Agent进入阻塞状态，用户无法继续对话，也看不到任务进度。

### 之前想的三层架构

```
Dispatcher（前台）→ Orchestrator（中控）→ Workers（干活）
```

- **Dispatcher**：只收任务、排队、叫号。prompt简单，不会被历史上下文污染。可以随时问"还剩多少任务"或"T-005进度怎么样"
- **Orchestrator**：只处理Dispatcher投来的1-3个任务。拆解→分配→验收。不直接接收人类输入，保持上下文干净
- **Workers**：独立session的专职Bot。被@到就干活，完成后发回中控。可以用不同模型

### 解决的核心问题

1. 上下文不会爆炸——中控每次只处理少量任务
2. 任务可以持久化——队列机制，慢慢消化
3. 容错——Worker挂了可重试，中控卡了Dispatcher报警
4. 用户不阻塞——随时可以和Dispatcher对话查看进度

这个架构是否就是解决吴律"没反应"的答案？但实现它需要多久？能不能先用临时方案稳住吴律，再逐步改造？

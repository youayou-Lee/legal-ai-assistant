# 2026-04-07 思考

## 新认知：幻觉是AI全链路问题，不是某个环节的问题

**触发事件**：今天开发完法条WebSearch验证机制后，我（Claude）在对比AI版和律师版上诉状时，在对比表里编造了律师版使用"证据5"这个事实。律师版根本没有写"证据5"，是直接描述证据内容。用户当场抓住了。

**这说明了什么**：

1. 幻觉不只是GLM-5的"法条引用错误"问题——它是**所有AI的系统性问题**
2. 我刚刚开发了法条WebSearch验证来解决这个问题，但**我自己就在做分析时犯了同样的错误**
3. 问题是：不确定什么时候AI会自信地编造东西。这次是在对比分析里，下次可能在哪里？
4. 这比之前认知的更严重——不只是"生成文书时法条编造"，而是"AI输出的任何事实性陈述都可能编造"

**与历史认知的关系**：

- 阶段九说"推理纪律缺失 ≠ 专业知识缺失"——部分正确，但幻觉问题比"纪律"更根本，是LLM的架构特性
- 阶段十说"按类型分别诊断"——但幻觉是跨类型的，不区分合同/婚姻/劳动
- 这次认知修正：**幻觉是比"推理纪律"和"知识缺失"更底层的风险**

**对产品的影响**：

- 法条WebSearch验证只解决了"法条引用"这一个点
- 但AI输出的其他事实性内容（证据描述、案件事实、当事人陈述）也可能编造
- 完全消除幻觉在当前技术下不可能——LLM就是会编
- 现实的策略是：**让律师知道哪些可信、哪些需要核实**，而不是试图消除所有幻觉

**可能的应对方向**（待深入讨论）：

1. **标注信心等级**：AI输出时标注"来自原始材料"vs"AI推理/总结"，让律师知道哪些需要重点核实
2. **事实溯源**：每个关键事实性陈述附上出处（哪份材料第几页）
3. **用户教育**：明确告知律师AI会编东西，养成核实习惯
4. **多模型交叉验证**：不同模型同时生成，对比差异——差异大的地方就是高风险点

**核心认知**：不要信任AI的任何事实性陈述，除非有原始材料支撑。这个原则适用于产品里的GLM-5，也适用于现在跟你对话的我。

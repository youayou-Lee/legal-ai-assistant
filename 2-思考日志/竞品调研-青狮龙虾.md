# 青狮龙虾（QingClaw）调研报告

> 调研日期：2026-04-02
> 来源：https://qingclaw.com/ 及相关公开报道
> 关联：[[德恒合伙人拿下方案]]、[[2026-04-02]]

## 一、产品本质

**青狮龙虾 = OpenClaw + 16项法律场景Skill + 培训服务**

和我们的产品技术底座完全相同（都基于OpenClaw），差异化在：
- 他们做了16项预配置Skill（合同审核、文书撰写、公众号写作、PPT制作等）
- 主打"降低使用门槛"——律师不需要懂技术
- 核心收入来自培训而非软件本身

## 二、商业模式

| 层级 | 内容 | 定价 |
|------|------|------|
| 产品层 | OpenClaw定制版 + 16项Skill | 包含在课程价格中 |
| 培训层 | 线上课程/线下训练营/团队内训 | 1999-2999元/人 |
| 服务层 | 社群答疑、会议答疑、远程服务 | 包含在课程中 |

**核心公式**：开源工具 × 垂直场景 × 培训变现 = 轻资产高利润

## 三、可借鉴的打法

### 1. "产品+培训"模式值得学

产品效果严重依赖用户使用水平（怎么提问、怎么拆任务、怎么审AI输出）。光卖工具不够，必须教人用。

### 2. 降低门槛是核心

- 预配置Skill，律师不需要自己配置
- 视频教学替代文档
- 陪伴式学习替代一次性交付

### 3. 飞轮效应

```
工具普及 → 场景化定制 → 培训变现 → 用户成功案例 → 口碑传播 → 更多用户
```

### 4. 个人IP + 社群运营

创始人镐赞定位"技术派律师"，知乎/公众号持续输出，700+学员社群。

## 四、关键风险发现：德恒已发文警告OpenClaw

**德恒律师事务所已发布文章《律师该不该"养龙虾"？》，核心观点：**

1. OpenClaw需要高权限（文件系统、邮件、通讯录），与律师保密义务冲突
2. 调用境外大模型可能构成数据出境
3. AI自主执行产生不良后果时责任归属不清
4. 26.1%的智能体技能存在安全漏洞
5. **结论：现阶段OpenClaw不适合直接进入律师行业真实业务场景**

**这对我们的德恒方案有重大影响。** 详见 [[德恒合伙人拿下方案]] 更新。

## 五、与我们的差异

| 维度 | 青狮龙虾 | 我们 |
|------|---------|------|
| 技术底座 | OpenClaw | OpenClaw（相同） |
| 目标用户 | 全体律师（通用） | 刑辩+民事律师（垂直） |
| 核心价值 | 教律师用AI | 帮律师分析案件（服务式） |
| 变现方式 | 培训收费 | 待定（可能是服务/合同/培训组合） |
| Skill方向 | 通用办公（PPT、公众号、排版） | 法律专业（案件分析、文书生成） |
| 数据安全 | 未重点强调 | 必须作为核心卖点 |

## 六、对我们的启示

### 必须做的

1. **数据安全必须成为第一卖点** — 德恒已经警告过OpenClaw的风险，如果我们不能解决这个问题，根本没有机会
2. **本地化部署方案** — 强调数据不出本机/律所服务器
3. **区别于青狮** — 他们教律师用AI，我们帮律师做分析。定位完全不同

### 可以学的

1. 产品+培训的组合模式
2. 预配置降低门槛
3. 陪伴式学习/服务
4. 个人IP建设

### 差异化方向

我们不是"教律师养龙虾"，而是"帮律师分析案件"。
青狮 = 工具培训（教你怎么用锤子）
我们 = 专业服务（帮你用锤子盖房子）

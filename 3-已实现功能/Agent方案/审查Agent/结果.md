## v1 vs v2 版本对比表

| 指标 | v1 | v2 | 改进原因 |
|------|----|----|---------|
| 预设缺陷数 | 3个 | 5个 | 测试复杂度+67% |
| 缺陷类型覆盖 | 1种（金额） | 5种（金额/法条/证据/请求/逻辑） | 多维度验证 |
| 总发现率 | 100% | 100% | 持平 |
| **共识发现率** | **100%** | **100%** | **v1→v2: D2从单点→共识** |
| 误报数 | 0个 | 0个 | 实务容忍度生效 |
| 检查清单 | 泛泛指令 | 结构化6项 | 按文书类型专项检查 |
| 闭环能力 | 无（仅告知） | 5步修正协议 | 自动修正+升级判断 |
| 冲突处理 | 无规则 | 取高不取低 | D3冲突成功解决 |

### 关键发现

1. **结构化检查清单提升共识率**：D2（法条遗漏）在v1中仅Qwen单点发现，v2因DeepSeek增加了"必须引用的特别法"子项而成为共识发现
2. **实务容忍度消除误报**："多次催要"在v1中被标记为缺陷，v2因实务容忍度说明被正确过滤
3. **取高不取低解决冲突**：D3在两方严重程度判断不同时，采用更保守的🔴致命，符合律师防御性思维
4. **闭环修正可行**：5个问题均可通过明确的修正操作解决，无需人工判断修不修（只判断修得对不对）

### 仍存在的问题

- 模拟≠真实：v2测试是人工模拟审查过程，真实AI模型行为可能有偏差
- 单一案例：仅测试了民间借贷起诉状一种输入
- 闭环未真实执行：自动修正的质量（尤其是金额重算）需律师确认

# 上诉状生成Skill（civil-appeal）

> 民事二审上诉状自动生成，7步强制推理。

## 部署位置

`~/.agents/skills/civil-appeal/SKILL.md`

## 设计思路

吴律用杨刚案测试后发现，无Skill约束时GLM-5生成的上诉状和律师期望差距大：
- 论证顺序不对（应优先程序违法，而非对方过错）
- 编号体系混乱（混用中文/阿拉伯数字）
- 证据引用模糊（用"证据5"而非具体描述）
- 论证罗列所有可能点而非聚焦

参考杨刚案律师修订版的结构特征，设计7步强制推理流程。

## 核心流程（7步）

1. **拆解一审判决** — 提取当事人、诉讼请求、法院认定事实、判决结果、法律依据
2. **错误识别** — 按严重程度排序：程序错误 > 实体错误 > 计算错误
3. **确定上诉请求** — 对应一审判项，明确每项请求（维/撤/改/驳）
4. **构建论证结构** — 按错误严重程度排序论证，非按发现顺序
5. **法条WebSearch验证** — 强制联网验证每条法条引用
6. **撰写上诉状** — 按模板输出，遵循文书质量约束
7. **自检+输出docx** — 自检清单（结构/编号/证据引用/法条/格式/法言法语）

## 文书质量共享约束

与起诉状、答辩状统一的约束：
- 证据引用：必须包含编号+内容摘要
- 编号体系：一→1.→首先/其次，禁止混用
- 格式：禁止横线、Markdown符号、裸段落
- 法言法语：禁止口语化/情绪化/胜率预测
- 法条验证：生成前必须WebSearch验证

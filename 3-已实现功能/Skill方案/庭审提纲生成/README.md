# 庭审提纲生成Skill

## 技术方案
OpenClaw Skill（`SKILL.md` + 6步强制推理框架）

## 一句话说明
输入案件材料和己方立场，按"五张纸"结构输出模块化庭审提纲，律师可分别打印带去法庭。

## 部署位置
`~/.agents/skills/trial-outline/`

## 核心设计洞察

1. **庭审提纲不是文书，是决策备忘录** — 律师在庭上时间有限，需要按庭审流程排列的速查表
2. **模块化输出（五张纸）** — 事实与法律关系、诉请/答辩要点、举证质证要点、争议焦点与预案、发问与法官提问，可分别打印
3. **独立运行** — 内部包含预判逻辑，不需要先跑opponent-prediction再跑庭审提纲
4. **争议焦点≤3个** — 调研显示超过3个焦点不再是"命脉"，非核心点放质证或代理词

## 核心流程（6步）

0. **锁定立场与案件类型** — 确认原告/被告视角
1. **事实时间线提取** — 从材料提取关键事件，按时间排序
2. **诉请/答辩要点整理** — 逐条列明主张+法条依据+证据支撑
3. **举证质证要点** — 己方证据清单+证明目的 + 对方证据预判+质证预案
4. **争议焦点归纳** — ≤3个核心焦点 + 应对预案
5. **发问与法官提问** — 发问清单 + 法官提问预案
6. **自检** — 对照质量标准逐项检查

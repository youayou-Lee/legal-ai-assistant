# 庭审提纲生成后续方向

## 短期（1周内）

### 1. 被告视角测试

- 用被告立场重新测试模拟案例
- 验证Step 2答辩要点模板的输出质量
- 对比原告/被告视角的输出差异

### 2. 吴律确认

- 请吴律review模拟测试输出
- 确认"五张纸"模块化结构是否符合她的使用习惯
- 确认争议焦点≤3个的约束是否合理
- 获取1份真实庭审提纲作为参考格式

### 3. OpenClaw端到端验证

- 在OpenClaw中实际运行
- 验证法条WebSearch步骤
- 验证docx输出格式

## 中期（2-4周）

### 4. 扩展案件类型

- 合同纠纷专项测试
- 婚姻财产专项测试
- 劳动争议专项测试
- 每种类型补充争议焦点和发问重点

### 5. 口头表达风格优化

- 区分"书面分析"和"口头表达"的语言风格
- 质证意见和发问清单使用更接近口头表达的表述
- 法官提问预案保持书面风格

### 6. 与其他Skill联动

- 起诉状生成后提示"是否需要生成庭审提纲"
- 探索自动传递案件材料到庭审提纲Skill

## 长期（1个月+）

### 7. 真实案例验证

- 用已结案的真实案例测试
- 对比AI庭审提纲与律师实际庭审准备的差异
- 评估争议焦点预判的命中率

### 8. 个性化适配

- 根据律师反馈积累偏好
- 不同律师可能有不同的提纲组织习惯
- 考虑法官风格预判（如已知法官的庭审习惯）

## 优先级排序

| 优先级 | 任务 | 依赖 |
|--------|------|------|
| P0 | OpenClaw端到端验证 | 共性阻塞问题解决 |
| P0 | 吴律确认 | 吴律出差回来 |
| P1 | 被告视角测试 | 无 |
| P2 | 扩展案件类型 | P0完成 |
| P3 | 口头表达风格优化 | P0完成 |
| P4 | Skill联动 | 多个Skill稳定后 |

# 庭审提纲生成 - 测试案例与方法

---

## 📋 一键测试

> **用法**：复制下方框内全部内容，粘贴到 OpenClaw 对话框发送即可。

```
帮我生成庭审提纲：

【我的身份】原告方（出借人）

【案件事实】
2023年3月15日，被告张三以资金周转为由向我借款人民币200,000元，双方签订《借款协议》一份，约定：借款期限6个月（2023年3月15日至2023年9月15日），月利率1.5%，到期一次性还本付息。

同日，我通过银行转账向张三支付人民币200,000元。张三出具收条一份。

2023年6月20日，张三通过微信向我还款人民币50,000元，双方未约定该还款抵充本金还是利息。

借款到期后，张三未归还剩余款项。我于2023年10月、11月多次通过微信催讨，张三以各种理由推脱。

2024年3月15日，我委托律师发送律师函催讨，张三签收后仍未还款。

我于2024年6月1日向法院提起诉讼。

【我的诉讼请求】
1. 判令被告偿还借款本金及利息
2. 诉讼费用由被告承担

【我方的证据】
1. 《借款协议》原件
2. 银行转账凭证
3. 被告出具的收条
4. 微信聊天记录（含催款记录和还款记录）
5. 律师函及签收凭证
```

**验证重点**：
1. 身份识别：应自动识别"我=原告方"
2. 时间线：应提取8个关键节点（签约→转账→收条→还款→到期→催讨→律师函→起诉）
3. 利率合法性：月利率1.5%=年化18%，应检查是否超过LPR四倍（约14.6%），并标注超限
4. 争议焦点：应≤3个，应包含还款抵充顺序和利率上限
5. 发问清单：应全部为封闭/引导式（"是否...""有没有..."），每个问题有明确目的
6. 最后陈述：应不超过3句话
7. 6个模块全部输出，每个模块可独立使用

## 测试目标

验证庭审提纲Skill能否按"五张纸"结构输出完整的模块化庭审提纲，重点验证：
1. 事实提取的完整性
2. 法条引用的准确性
3. 争议焦点归纳的精准性（≤3个）
4. 质证预案的覆盖度
5. 发问清单的实用性（封闭式+有目的）

## 测试案例：民间借贷纠纷（原告方视角）

### 案件基本信息

**案由**：民间借贷纠纷
**己方立场**：原告方
**案件类型**：民间借贷

### 案件事实

**借款基本信息**：
- 借款金额：人民币200,000元
- 月利率：1.5%（年化18%）
- 借款期限：6个月（2023年3月15日至2023年9月15日）
- 还款方式：到期一次性还本付息

**关键事件**：
1. 2023年3月15日：签订借款协议 + 银行转账200,000元 + 收条
2. 2023年6月20日：被告微信还款50,000元（未约定抵充本金/利息）
3. 2023年9月15日：借款到期，被告未还
4. 2023年10-11月：原告微信催讨，被告推脱
5. 2024年3月15日：律师函催告，被告签收
6. 2024年6月1日：起诉

### 设计的风险点

| 风险点 | 设计意图 | 预期AI应发现 |
|--------|---------|-------------|
| 月利率1.5%=年化18% | 超过LPR四倍（约14.6%） | 利率合法性检查应标注超限 |
| 还款50,000元未约定性质 | 抵充顺序争议 | 应分析先息后本vs先本后息 |
| 微信催讨 | 诉讼时效中断证据 | 应确认未超时效 |
| 到期后9个月才起诉 | 是否怠于行使权利 | 法官可能追问 |

### 证据清单

| 编号 | 证据名称 | 形式 | 证明目的 |
|------|---------|------|---------|
| 1 | 《借款协议》 | 书证（原件） | 借贷关系成立+约定内容 |
| 2 | 银行转账凭证 | 书证（银行盖章件） | 实际交付200,000元 |
| 3 | 被告收条 | 书证（原件） | 被告确认收款 |
| 4 | 微信聊天记录 | 电子数据 | 还款事实+催讨事实 |
| 5 | 律师函+签收凭证 | 书证 | 诉讼时效中断 |

## 验证清单

### Step 0: 身份识别

- [ ] 己方立场正确识别为原告方
- [ ] 案件类型识别为民间借贷纠纷

### Step 1: 事实时间线

- [ ] 包含签约日期（2023年3月15日）
- [ ] 包含转账交付事实
- [ ] 包含被告还款50,000元（2023年6月20日）
- [ ] 包含借款到期（2023年9月15日）
- [ ] 包含催讨事实（2023年10-11月）
- [ ] 包含律师函（2024年3月15日）
- [ ] 包含起诉（2024年6月1日）
- [ ] 法律关系说明准确

### Step 2: 诉请/答辩要点

- [ ] 诉讼请求逐条列明
- [ ] 每条请求有法条依据（具体到法律名称+条款号）
- [ ] 每条请求有证据支撑
- [ ] **利率合法性检查**：标注月利率1.5%是否超过LPR四倍
- [ ] 利息计算方式说明（先息后本抵充规则）

### Step 3: 举证质证

- [ ] 己方5份证据全部列明
- [ ] 每份证据有证明目的
- [ ] 每份证据预判了对方可能质证
- [ ] 对方证据质证预案按"三性"（真实性/合法性/关联性/证明力）组织
- [ ] 质证表述使用法言法语（不只是"不予认可"）

### Step 4: 争议焦点

- [ ] 争议焦点≤3个
- [ ] 包含还款抵充顺序争议
- [ ] 包含利率上限争议
- [ ] 每个焦点有：己方观点+法条+对方预判+反驳预案
- [ ] 每个焦点有法官可能追问+应对

### Step 5: 发问与法官提问

- [ ] 发问对象明确（被告张三）
- [ ] 每个发问有明确目的
- [ ] 发问为封闭/引导式（"是否...""有没有..."而非"为什么..."）
- [ ] 法官提问预案覆盖每个争议焦点
- [ ] 法官提问的回答有具体要点

### Step 6: 最后陈述

- [ ] 不超过3句话
- [ ] 包含核心诉求
- [ ] 包含收尾请求

### Step 7: 自检

- [ ] 全文法言法语（无口语化表述）
- [ ] 金额格式："人民币XX,XXX元"
- [ ] 日期格式："XXXX年XX月XX日"
- [ ] 法条引用具体到条款号
- [ ] 无概率/胜率数字

## 评分标准

### 评分维度

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| **完整性** | 25% | 6个模块全部覆盖，事实无遗漏 |
| **实用性** | 25% | 输出可直接打印带去法庭 |
| **精准性** | 20% | 争议焦点准确，法条正确 |
| **对抗性** | 15% | 预判对方策略的覆盖度 |
| **格式规范** | 15% | 表格化、法言法语、金额日期格式 |

### 具体评分细则

#### 完整性（25分）

- **优秀（23-25分）**：6模块全覆盖，时间线8节点无遗漏，利率合法性检查到位
- **良好（20-22分）**：6模块全覆盖，少量细节遗漏
- **及格（15-19分）**：5个模块覆盖，1个模块缺失
- **不及格（0-14分）**：多个模块缺失或严重不完整

#### 实用性（25分）

- **优秀（23-25分）**：表格为主，信息密度高，可直接打印使用；发问为封闭式
- **良好（20-22分）**：基本可打印使用，少量格式问题
- **及格（15-19分）**：需要修改后才能使用
- **不及格（0-14分）**：无法直接使用，需要大幅重写

#### 精准性（20分）

- **优秀（18-20分）**：争议焦点≤3个且全部准确；法条引用正确；利率检查正确
- **良好（15-17分）**：焦点基本准确，个别法条引用不精确
- **及格（12-14分）**：焦点超过3个或遗漏重要焦点
- **不及格（0-11分）**：焦点归纳完全不准或法条编造

#### 对抗性（15分）

- **优秀（14-15分）**：对方策略预判全面，质证预案按三性准备，反驳具体
- **良好（12-13分）**：主要预判到位，部分遗漏
- **及格（10-11分）**：预判过于笼统
- **不及格（0-9分）**：缺乏对抗性分析

#### 格式规范（15分）

- **优秀（14-15分）**：全文表格化，法言法语，金额/日期格式统一
- **良好（12-13分）**：基本规范，少量口语化表述
- **及格（10-11分）**：格式基本可接受
- **不及格（0-9分）**：格式混乱，大量口语化

## 边界测试场景

### 场景1：被告方视角

**测试目的**：验证被告视角的庭审提纲质量

**预期差异**：
- Step 2应为答辩要点而非诉讼请求
- 发问对象为原告而非被告
- 质证方向相反（攻击原告证据而非预判被告攻击）
- 答辩策略完全不同

### 场景2：无争议焦点

**输入**：被告完全承认借款事实和金额，仅主张经济困难请求分期

**预期输出**：
- 争议焦点可少于3个
- 重点转向调解/分期方案
- 发问聚焦还款能力

### 场景3：证据极少

**输入**：仅有口头约定，无书面协议，无转账记录

**预期输出**：
- 标注证据链严重不完整
- 争议焦点应包含"借贷关系是否成立"
- 建议补充证据

### 场景4：合同纠纷（非借贷）

**输入**：买卖合同纠纷，被告未按约定交货

**预期输出**：
- 案件类型正确识别为合同纠纷
- 争议焦点围绕违约责任、损失计算
- 不应出现借贷专用的利率合法性检查

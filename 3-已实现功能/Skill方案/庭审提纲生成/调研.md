# 庭审提纲生成背景调研

## 调研目标

了解律师庭审准备的实际工作流程和庭审提纲的实务标准，明确Skill的价值定位和设计方向。

## 调研方法

WebSearch搜索3组关键词：
1. "律师庭审准备 庭审提纲 标准流程 实务经验"
2. "庭审提纲 模板 争议焦点 举证质证 发问提纲 律师实务"
3. "民事庭审 律师准备 争议焦点归纳 法官发问 庭审技巧"

## 调研发现

### 发现1：庭审提纲是律师最重要的庭前准备文件

多个实务来源一致指出：**庭审提纲是律师庭审准备的核心产出物**。

> "庭审提纲是律师围绕庭审程序，针对案件事实及争议焦点进行归纳、预演后形成的书面材料。其核心目的是保障庭审质量和效率。"

**为什么重要**：
- 律师在庭审中时间有限（简易程序可能只有30分钟），需要快速查阅准备好的材料
- 庭审按固定流程进行，律师需要在每个环节都有预案
- 庭审中可能遇到对方"突袭"，需要预先准备应对方案

**来源**：[知乎 - 诉讼实务：民事诉讼案件庭前准备清单](https://zhuanlan.zhihu.com/p/671822209)、[网易 - 开庭提纲必备的5个关键点](https://c.m.163.com/news/a/K3V34ONV05455BAL.html)

### 发现2："五张纸法则"是行业标准

多个资深律师的经验总结：出庭律师的庭审准备可以浓缩为"五张纸"：

| 纸 | 内容 | 用途 |
|----|------|------|
| 第一张 | 案件事实时间线 | 庭审陈述时参照 |
| 第二张 | 诉讼请求/答辩要点 + 法律依据对照表 | 法庭调查环节 |
| 第三张 | 举证质证要点一览表 | 举证质证环节 |
| 第四张 | 争议焦点预判 + 应对方案 | 法庭辩论环节 |
| 第五张 | 发问提纲 + 法官可能提问的预案 | 法庭发问环节 |

**关键启示**：模块化输出不是我的发明，而是律师实务中的标准做法。分模块输出比一份完整文档更实用——律师可以分别打印，庭审时按环节翻阅。

**来源**：[知乎 - 律师如何写好庭审提纲](https://zhuanlan.zhihu.com/p/162785197)、[搜狐 - 庭审提纲的妙用技巧](https://www.sohu.com/a/483418397_121123817)

### 发现3：争议焦点不宜超过3个

> "超过3个以上的争议焦点，这个焦点其实已经不是案件的'命脉'，其他的非关键性重要的争议点可以适当在质证意见中进行陈述，当然也可以在补充在庭后向法庭提交的补充代理意见。"

**关键启示**：争议焦点是整份庭审提纲的"锚点"，其他部分都围绕它展开。限制焦点数量迫使律师（和AI）聚焦核心矛盾，避免"什么都想说，什么都说不清"。

**来源**：[知乎 - 律师如何写好庭审提纲](https://zhuanlan.zhihu.com/p/162785197)

### 发现4：庭审提纲按开庭流程排列

**最常见且实用的庭审提纲范式是按照开庭流程依次罗列**：

1. 身份信息核实
2. 开庭陈述（原告诉请/被告答辩）
3. 法庭调查（举证质证）—— **重点环节**
4. 争议焦点归纳与回应
5. 法庭发问
6. 法庭辩论
7. 最后陈述

> "法庭调查是庭审的核心环节，律师需要围绕争议焦点组织证据目录，注意举证责任分配。"

**来源**：[网易 - 开庭提纲必备的5个关键点](https://c.m.163.com/news/a/K3V34ONV05455BAL.html)

### 发现5：发问提纲必须提前设计，庭上调整

律师在庭前一定要准备一份**发问提纲**：

- 如果有多名发问对象，更要根据不同对象分别准备
- 通过发问提纲可以将庭审中需要通过发问揭示的证据矛盾、事实疑点等系统化整理
- 发问要**围绕争议焦点和调查重点**，同时关注双方陈述、举证质证等环节遗漏的问题
- **发问必须有明确目的**，避免开放式问题给对方"表演"空间

**关键启示**：发问不是即兴发挥，而是有目的的证据揭示工具。AI生成的发问清单必须是封闭/引导式，每个问题都要标注目的。

**来源**：[无讼 - 民事诉讼中律师庭审发问技巧](https://victory.itslaw.com/html/detail/251ea83f-f6e5-4f28-8c7e-f05dd52e6533/)

### 发现6：争议焦点归纳有两个时间点

> "争议焦点的确定有两个时间点：一是开庭前通过阅卷，认真审查双方当事人诉辩要点，列出书面庭审提纲；二是开庭时引导当事人围绕争议焦点进行辩论。"

**关键启示**：AI生成的庭审提纲是"庭前预判"，开庭后律师需要根据庭审情况调整。这不是缺陷——庭前预判本身就是庭审提纲的核心价值。

**来源**：[知乎 - 庭审争议焦点，法官建议这样归纳](https://zhuanlan.zhihu.com/p/565843092)、[江苏法院网 - 浅析民事案件中争议焦点的归纳](http://www.jsfy.gov.cn/article/12973.html)

### 发现7：庭前准备标准流程

实务中的庭前准备通常遵循以下流程：

1. **全面阅卷** — 认真审阅全部案卷材料，形成初步印象
2. **证据梳理与固定** — 组织证据目录，明确每份证据的证明目的；预判对方证据，准备质证预案
3. **法律检索** — 检索相关法律法规、司法解释
4. **制作庭审提纲** — 按庭审流程逐项准备
5. **庭前会议（如适用）** — 了解对方立场、固定争议焦点
6. **模拟演练** — 预演对方可能的攻击角度

**关键启示**：庭审提纲的制作是庭前准备的第4步，前面3步（阅卷、证据梳理、法律检索）是前提。在我们的系统中，这些前提由其他Skill完成（起诉状/答辩状/预判模块），庭审提纲Skill负责第4步的整合。

**来源**：[松阳县人民法院 - 浅谈民事诉讼庭前准备工作](https://sy.lsfy.gov.cn/dwfc/xszc/2024-05-18/28394.html)

## 质证环节的实务标准

### 质证按"三性"逐项进行

律师质证时必须围绕证据的三个维度：

| 维度 | 含义 | 典型质证话术 |
|------|------|-------------|
| **真实性** | 证据是否伪造、变造、篡改 | "对该证据的真实性不予认可，该文件存在涂改痕迹" |
| **合法性** | 取证方式是否合法 | "该录音系偷录取得，取证方式不合法" |
| **关联性** | 证据与待证事实之间的联系 | "该证据与本案待证事实无关" |
| **证明力** | 承认真实性但不认可证明目的 | "真实性无异议，但不证明对方主张的XX事实" |

### 质证的常见策略

1. **直接否认真实性**：适用于对方无法提供原件的情况
2. **质疑取证合法性**：适用于偷录偷拍等证据
3. **切断关联性**：承认真实性，但主张与本案无关
4. **限制证明力**：最常用的策略——"真实性无异议，但不证明对方主张的XX事实"

**关键启示**：庭审提纲的质证预案必须按"三性"逐项准备，且必须提供具体的质证表述（不只是"不予认可"）。

## 律师对庭审提纲工具的需求

### 需求1：不遗漏

律师最怕的不是"不知道怎么打"，而是"没想到对方会出这招"。

> ——吴律需求文档原话

庭审提纲的核心价值是**确保每个环节都有预案、不遗漏**。AI的系统化推理框架可以帮助律师避免遗漏关键风险点。

### 需求2：争议焦点+发问清单

吴律在需求文档中明确列出：

> "AI帮我做：总结争议焦点及法官发问清单"
> "AI帮我做：生成庭审提纲"

### 需求3：不要过程，要结果

> "过多展示过程性的东西"——吴律标注为"无必要"

律师不需要看AI的推理过程，只需要可用的庭审准备材料。

## 与项目的关系

### 庭审提纲在案件流程中的位置

```
案件材料 → [起诉状/答辩状生成] → [预判模块] → [庭审提纲生成] → 庭审
                    ↓                    ↓                ↓
              文书+证据清单      对方策略预判      按庭审流程整合
```

庭审提纲是**庭前准备的最终整合环节**，将前面所有Skill的产出按庭审流程重新组织。

### 庭审提纲的独特价值

与其他Skill相比，庭审提纲的独特价值在于：

1. **整合而非生成**：不需要从零产生新信息，而是按庭审流程重组已有信息
2. **对抗性**：不仅准备己方材料，还要预判对方策略并准备应对
3. **面向实战**：输出是给律师在法庭上直接使用的速查表，不是法律分析报告

## 调研结论

1. **"五张纸法则"是行业标准** — 模块化输出符合律师实际工作习惯
2. **争议焦点≤3个** — 聚焦核心矛盾，避免信息过载
3. **发问清单必须有目的** — 封闭/引导式问题，不给对方"表演"空间
4. **质证按三性准备** — 真实性、合法性、关联性、证明力四维度
5. **庭审提纲是整合环节** — 不是从零生成，而是按流程重组
6. **律师要的是速查表不是论文** — 表格化、简练、每句话有用

这些结论直接指导了庭审提纲Skill的设计。

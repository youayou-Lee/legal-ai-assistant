# 权益份额精算

## 技术方案
OpenClaw Skill（`SKILL.md` + 5步强制推理 + 3大思维陷阱防护）

## 一句话说明
防止份额计算中的思维陷阱（静态思维、片段思维、锚定效应），强制先建时间线再计算。

## 部署位置
`~/.agents/skills/legal-analysis-quality/`

## 问题来源

黄律继承份额分析反馈——AI分析"不对"，应先确认遗产继承问题再算份额（推理顺序错误）。

详细诊断发现：
- 这不是孤例，是通用Agent法律推理能力的系统性问题（4/4确认）
- 继承类型经详细诊断：70%推理纪律缺失 + 30%知识缺失
- 本 Skill 专注于解决"推理纪律"层面

## 核心机制

### 三大思维陷阱

| 陷阱 | 表现 | 示例 |
|------|------|------|
| 静态思维 | 只看当前登记份额，忽略历史变化 | 看到"1/3"直接算，不追问来源 |
| 片段思维 | 只看单一时点，不建时间线 | 遗漏中间死亡事件导致份额流转 |
| 锚定效应 | 被显性数字锁定停止追问 | 看到房产证70%就不深究共有关系变化 |

### 5步强制推理

1. **建时间线**（禁止任何计算）— 梳理所有时间节点和法律事件
2. **列权益来源**（仍然禁止计算）— 取得方式、变更原因、变动依据
3. **确定计算规则** — 适用法律规则、计算基准点、参数
4. **计算**（前三步验证后才能开始）— 完整性+时序性+关系性验证
5. **验证** — 份额加总=100%？逻辑可复现？假设条件已标注？

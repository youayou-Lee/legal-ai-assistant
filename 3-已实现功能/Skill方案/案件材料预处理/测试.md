# 案件材料预处理 - 测试

## 📋 一键测试

> **用法**：复制下方框内全部内容，粘贴到 OpenClaw 对话框发送即可。

```
请预处理案件材料，PDF目录在 /home/you/桌面/caseData/xbg/陈明飞诈骗案，案件名称是陈明飞诈骗案
```

**前提条件**：
- OpenClaw 网关已启动（`openclaw gateway restart`）
- PDF目录存在且包含PDF文件
- PaddleOCR API Token 有效

**验证重点**：
1. Skill被正确触发（输出应包含 `材料/`、`提取报告.json`、`材料目录.md`、`关系图谱.canvas`）
2. `材料/` 目录下每个PDF都有对应的 `.md` 文件
3. `提取报告.json` 显示无 failed 文件
4. `材料目录.md` 包含三段：材料位置索引 + 信息定位指引 + 案件概要
5. `关系图谱.canvas` 是合法JSON，包含人物/金额/日期/法律关系节点
6. 关系图谱中文件节点正确链接到 `材料/` 下的MD文件

## 测试案例

### 案例1：陈明飞诈骗案（刑事）

**基本案情**：被告人陈明飞谎称可办理入读锦兴小学，骗取被害人成龙飞42,000元，已退还18,000元。前罪招摇撞骗罪，数罪并罚。

**证据材料**：
- 起诉书（.wps格式）
- 文书卷：受案登记表、立案决定书、取保候审、起诉意见书
- 证据卷1：嫌疑人讯问笔录2次、辨认笔录、扣押笔录
- 证据卷2：被害人询问笔录2次、证人证言3份、辨认笔录
- 一审判决书

**风险点设计**：
- 含WPS格式文件（非PDF），验证脚本兼容性
- 证据卷单文件150万+字符，验证大文件处理
- 刑事案卷结构，验证文书类型识别准确性

### 案例2：杨刚案（民事借贷）

**基本案情**：原告杨江诉被告杨刚借贷纠纷，本金80万+利息。含鉴定报告、多份判决书。

**证据材料**：
- 起诉状2份（新旧版）
- 证据材料（含鉴定报告）
- 判决书2份
- 原告补充证据
- 原告补充鉴定报告机构及人员资质

**风险点设计**：
- 含2份扫描型PDF（鉴定报告），验证OCR触发
- 同一当事人多份同类文书（2份起诉状、2份判决书），验证去重/区分能力

## 验证清单

### Stage 1: 文本提取
- [ ] `提取报告.json` 已生成，total = 实际PDF数量
- [ ] failed = 0
- [ ] 随机抽查1-2个MD文件，关键信息（金额、日期、当事人）完整
- [ ] 扫描型PDF正确触发OCR（method = "ocr"）

### Stage 2: 材料目录
- [ ] `材料目录.md` 每个文件都有类型识别和摘要
- [ ] 信息定位指引覆盖核心信息需求（当事人、金额、时间线）
- [ ] 案件概要中案由、当事人、金额准确
- [ ] 关键时间节点完整

### Stage 3: 关系图谱
- [ ] `关系图谱.canvas` 是合法JSON
- [ ] 所有当事人都有对应的人物节点
- [ ] 每个金额都有上下文标注（本金/利息/赔偿）
- [ ] 关键法律关系（借贷/担保/诈骗等）有边连接
- [ ] 文件节点正确链接到 `材料/` 下的MD文件
- [ ] Canvas 在 Obsidian 中可正常打开

## 评分标准

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| 完整性 | 30% | 所有PDF提取成功，三阶段全部产出，无遗漏模块 |
| 精准性 | 25% | 金额、日期、当事人、案由准确无误，文书类型识别正确 |
| 实用性 | 20% | 材料目录能有效指引Agent定位信息，信息定位条目≥10条 |
| 图谱质量 | 15% | 实体/关系完整，布局合理，来源标注，文件节点链接正确 |
| 格式规范 | 10% | Markdown/YAML/Canvas格式正确，frontmatter完整 |

每个维度按4档评分：优秀(90%+)、良好(80%+)、及格(60%+)、不及格(<60%)

## 边界测试

| 场景 | 预期结果 | 实测结果 |
|------|---------|---------|
| 空目录（无PDF文件） | 脚本报错退出，提示检查路径 | 未测试 |
| 单文件>50页 | 提示用户确认是否OCR | 未测试 |
| PaddleOCR Token过期 | 标记OCR失败，回退pdftotext结果 | 未测试 |
| 纯扫描型PDF（全OCR） | 所有文件走OCR管线 | 杨刚案中2个扫描型已验证通过 |
| 已有部分MD的增量处理 | 只处理新增PDF，不覆盖已有 | 未测试 |

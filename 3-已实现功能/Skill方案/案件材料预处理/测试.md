# 案件材料预处理 - 测试

## 📋 一键测试

> **用法**：复制下方框内全部内容，粘贴到 OpenClaw 对话框发送即可。

```
请预处理案件材料，PDF目录在 /home/you/桌面/caseData/xbg/陈明飞诈骗案，案件名称是陈明飞诈骗案
```

**前提条件**：
- OpenClaw 网关已启动（`openclaw gateway restart`）
- PDF目录存在且包含PDF文件
- PaddleOCR API Token 有效

**验证重点**：
1. Skill被正确触发（不是Agent自创方案）
2. `材料/` 目录下每个PDF都有对应的 `.md` 文件
3. `提取报告.json` 显示无 failed 文件
4. `材料目录.md` 包含三段：材料位置索引 + 信息定位指引 + 案件概要
5. `关系图谱.canvas` 是合法JSON，包含人物/金额/日期/法律关系节点
6. 关系图谱中文件节点正确链接到 `材料/` 下的MD文件

## 测试案例

### 案例1：陈明飞诈骗案（刑事）
- **输入**：4 PDF + 1 WPS
- **特点**：刑事案卷结构（文书卷 + 证据卷），文本型PDF
- **预期**：全部走pdftotext，0 OCR

### 案例2：杨刚案（民事借贷）
- **输入**：8 PDF
- **特点**：混合型（6文本 + 2扫描），含鉴定报告、判决书等
- **预期**：6 pdftotext + 2 OCR

## 验证清单

- [ ] `提取报告.json` 已生成，total = 实际PDF数量
- [ ] failed = 0
- [ ] 随机抽查1-2个MD文件，关键信息（金额、日期、当事人）完整
- [ ] `材料目录.md` 每个文件都有类型识别和摘要
- [ ] `材料目录.md` 信息定位指引覆盖核心信息需求
- [ ] `关系图谱.canvas` 所有当事人都有对应节点
- [ ] 关系图谱中每个金额都有上下文标注
- [ ] 关系图谱中关键法律关系有边连接
- [ ] Canvas 在 Obsidian 中可正常打开

## 评分标准

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| 完整性 | 30% | 所有PDF提取成功，三阶段全部产出 |
| 精准性 | 25% | 金额、日期、当事人、案由准确无误 |
| 实用性 | 20% | 材料目录能有效指引Agent定位信息 |
| 图谱质量 | 15% | 实体/关系完整，布局合理，来源标注 |
| 格式规范 | 10% | Markdown/YAML/Canvas格式正确 |

# 法条引用审查（WebSearch验证）

> 生成法律文书前，强制联网验证每条法条引用是否准确。

## 问题背景

吴律4/7杨刚案测试发现GLM-5引用民诉法76条内容完全错误：
- AI写的：76条关于鉴定程序
- 实际内容：76条是"证人出庭"
- 正确应为：79条（申请鉴定与专门性问题）

根因：GLM-5默认Agent不联网时，法条内容靠训练数据"猜"，存在幻觉。智谱清言开联网搜索、Qwen、DeepSeek都能正确回答。

## 解决方案

在三个文书Skill（起诉状/答辩状/上诉状）中新增法条WebSearch验证步骤：
- 起诉状：Step 4c
- 答辩状：Step 4.6
- 上诉状：Step 4.5

### 验证流程

1. AI生成文书时列出所有引用的法条
2. 对每条法条执行`web_search`搜索法条内容
3. 比对搜索结果与AI引用的内容
4. 输出验证结果表：

```
| 法条 | AI引用内容 | 实际内容 | 结果 |
|------|-----------|---------|------|
| 民诉法79条 | 申请鉴定... | ... | ✅正确 |
| 民诉法76条 | 鉴定程序... | 证人出庭... | ❌错误→已修正 |
```

5. 标注为正确/错误/待核实
6. 错误的法条自动修正后写入文书

## 已集成到的Skill

- `~/.agents/skills/civil-complaint/SKILL.md` — Step 4c
- `~/.agents/skills/civil-defense-statement/SKILL.md` — Step 4.6
- `~/.agents/skills/civil-appeal/SKILL.md` — Step 4.5

## ⚠️ 验证状态（2026-04-08 全局盘点确认）

- **测试环境**：仅在 Claude Code 中模拟测试
- **OpenClaw 中实际运行**：❌ 从未验证 Agent 能正确调用 web_search
- **覆盖范围**：仅解决了"法条引用"这一个幻觉点，AI 输出的其他事实性内容仍可能编造

## 局限性

- 只解决了"法条引用"这一个幻觉点
- AI输出的其他事实性内容（证据描述、案件事实）仍可能编造
- 完全消除幻觉在当前技术下不可能
- 更通用的反幻觉方案：多模型集成验证（随机森林策略）— 待开发

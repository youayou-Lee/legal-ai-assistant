# 法院文书自动处理

## 基本信息
- **技术方案**：OpenClaw Skill（SKILL.md + 9步自动化流程）
- **一句话说明**：律师转发法院短信即自动下载文书、微信汇报、设置提醒
- **部署位置**：`~/.agents/skills/court-document-handler/`
- **当前状态**：v1已部署，`openclaw skills list` 显示 `✓ ready ⚖️ court-document-handler`，待真实法院短信验证完整流程

## 功能描述

律师经常收到法院送达短信（如广西法院短信平台），包含文书下载链接和案号信息。律师可能忘记处理或错过重要时限（答辩期、举证期、上诉期）。

**解决方式**：律师将法院短信复制发送给 OpenClaw（通过微信），Skill 自动完成全流程：

1. 识别短信内容并提取链接
2. 用 OpenClaw browser 访问法院送达平台
3. 输入手机后六位验证码登录
4. 获取文书列表并下载 PDF
5. 提取案件关键信息（案号、案由、法院、当事人等）
6. 通过微信发送结构化汇报 + PDF 原件
7. 根据文书类型自动设置定时提醒（传票→开庭前1天，判决书→上诉截止前，举证通知→截止前3天）

**为谁解决**：主要服务使用广西法院电子送达平台的律师

**解决程度**：全自动处理，无需人工干预。目前仅支持广西法院短信平台。

## 实现架构

### 触发机制

Skill 通过关键词匹配自动触发，匹配特征：
- 发送方：`广西法院短信平台` / `法院向您发送`
- 链接特征：`sfsdw` / `文书详见`
- 案号格式：`(20XX)桂XXXX民初XXXX号`

### 核心流程

```
收到短信 → 正则提取链接 → openclaw browser start → openclaw browser open → 输入验证码(886288)
→ 获取文书列表 → 下载PDF → pdftotext提取内容 → 解析关键信息
→ openclaw message send 微信汇报+PDF → openclaw cron add 设置提醒 → openclaw browser stop
```

### 关键设计决策

1. **必须用 browser 不用 web_fetch**：法院系统需要验证码登录，web_fetch 无法处理交互式页面
2. **固定验证码**：手机后六位 886288，写入配置（当前为硬编码）
3. **提醒规则按文书类型区分**：传票/判决书/应诉通知书/举证通知书各有不同的提醒时机
4. **微信双发送**：先发结构化汇报消息，再发 PDF 原件，确保律师两样都能收到

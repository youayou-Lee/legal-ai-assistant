# 法院文书自动处理 — 后续方向

## 短期（1-2周）

1. **用真实法院短信端到端验证**
   - 等律师提供一条有效的法院送达短信
   - 验证全链路：浏览器访问→验证码→下载PDF→解析→微信发送→提醒设置
   - 验证完成后更新 [[结果]]

2. **验证码和微信账号提取为配置项**
   - 在 SKILL.md 中用变量替代硬编码值
   - 方便多律师使用时快速配置

## 中期（1-2月）

3. **多省份法院平台支持**
   - 调研广东、北京、上海等主要省份的法院电子送达平台格式
   - 提取通用模式，支持不同 URL 格式和案号格式
   - 验证方式统一为"手机后六位"（全国统一规则）

4. **OCR 增强 PDF 解析**
   - 对扫描件 PDF 使用 OCR 提取文字
   - 与项目中"OCR方案升级"排队任务联动（银行流水扫描件也是同类问题）

5. **与案件管理联动**
   - 下载的文书自动归档到对应案件目录
   - 使用 [[case-manager]] Skill 管理案件状态
   - 传票信息自动更新案件的开庭时间

## 长期（3-6月）

6. **时效管理看板**
   - 所有案件的时限汇总展示（答辩截止、举证截止、开庭日期、上诉截止）
   - 按紧急程度排序，即将到期的高亮提醒
   - 通过微信定期推送时效提醒汇总

7. **支持律师主动查询**
   - 律师发"这个案件还有什么期限"→ AI 汇总该案件的所有待办时限
   - 律师发"下周有哪些开庭"→ AI 汇总近期开庭安排

## 与其他功能的联动计划

| 联动功能 | 联动方式 | 优先级 |
|---------|---------|--------|
| [[../起诉状生成]] | 收到起诉状时自动触发起诉状分析 | 中 |
| [[../答辩状生成]] | 收到应诉通知书时提醒生成答辩状 | 高 |
| [[../../Agent方案/审查Agent]] | 下载的文书自动进入审查流程 | 低 |
| case-manager | 文书信息自动录入案件管理 | 中 |

> **短期方向对应 [[存在的问题]]**：P0（真实验证）→ 短期#1，P1（硬编码）→ 短期#2

# 法院文书自动处理 — 存在的问题

## 问题清单

### P0 — 阻塞性问题

1. **完整流程未经真实法院短信验证**
   - 当前仅在 Skill 加载层面验证通过（`openclaw skills list` 显示 ready）
   - 浏览器访问真实链接、验证码登录、下载 PDF、内容解析、微信发送、提醒设置——全链路未经真实数据验证
   - 需要律师提供一条有效的法院短信进行端到端测试

### P1 — 重要问题

2. **仅支持广西法院短信平台**
   - 链接正则只匹配 `http://171.106.48.55:28083/sfsdw//r/`
   - 案号正则只匹配 `(20XX)桂XXXX民初XXXX号`
   - 其他省份法院的短信格式、链接格式、验证方式可能不同
   - 需要逐省份调研电子送达平台差异

3. **验证码硬编码**
   - 当前验证码 `886288` 写死在 SKILL.md 的 Workflow 中
   - 仅适用于当前绑定的手机号
   - 如果律师更换手机号，需要手动修改 Skill

4. **PDF 内容解析依赖 pdftotext**
   - 法院文书有扫描件 PDF（图片型），`pdftotext` 无法提取文字
   - 需要引入 OCR 能力处理扫描件
   - 与项目排队任务"OCR方案升级"是同一类问题

### P2 — 一般问题

5. **提醒时间固定为 09:00**
   - 所有提醒都在早上 9 点触发
   - 未考虑律师个人偏好（有人习惯早上看消息，有人习惯晚上）
   - 传票提醒"开庭前1天09:00"如果开庭时间是上午，律师准备时间可能不够

6. **错误处理不够健壮**
   - 链接无法访问时只发一条微信通知
   - 没有重试机制（当前只重试验证码1次）
   - 没有记录失败日志，不利于问题排查

7. **微信账号硬编码**
   - `e1a1470017fa-im-bot` 和 `o9cq803rIUlhkxQniI8yry_atp88@im.wechat` 写死在 Skill 中
   - 换律师或换微信账号需要手动修改 SKILL.md

### P3 — 低优先级

8. **链接有效期6个月未利用**
   - 短信提到"文书链接6个月内有效"，Skill 没有保存链接用于后续查询
   - 可以考虑保存链接+案件映射，方便律师后续查阅

9. **不支持批量处理**
   - 一次只处理一条短信
   - 律师有时会同时收到多条短信（一个案件多份文书），需要逐条触发

# 法院文书自动处理 — 测试

---

## 📋 一键测试

> **用法**：复制下方框内全部内容，通过微信发送给 OpenClaw 即可。

```
【广西法院短信平台】张三：南宁市青秀区人民法院向您发送了（2026）桂0103民初1234号案件相关文书，文书详见:http://171.106.48.55:28083/sfsdw//r/TestABC123
。查看并下载您的文书，文书链接6个月内有效请尽快查阅下载。如链接无法打开查阅，请联系（集约送达中心）0771-12368
```

**验证重点**：
1. OpenClaw 是否触发 court-document-handler Skill（而非用 WebSearch 回复）
2. 是否启动浏览器并尝试访问链接（因测试链接无效，预期会触发错误处理流程）
3. 错误处理时是否通过微信通知用户"链接无法访问，请手动处理"
4. `openclaw skills list` 中 court-document-handler 是否显示 `✓ ready`

## 测试案例1：Skill 加载与触发验证

### 案件背景
模拟律师收到广西法院送达短信，验证 Skill 是否被正确触发。

### 输入材料
一条完整的法院短信文本（见上方一键测试）。

### 测试方法
1. 执行 `openclaw skills list`，确认 court-document-handler 显示 `✓ ready`
2. 通过微信向 OpenClaw 发送模拟短信
3. 观察 AI 响应行为：是触发 Skill 还是走默认 WebSearch

### 验证点清单

| # | 验证项 | 预期结果 | 实际结果 | 通过 |
|---|--------|---------|---------|------|
| 1 | Skill 加载 | skills list 显示 ready | ready | ✅ |
| 2 | 短信识别 | 匹配"广西法院短信平台"关键词 | 匹配 | 待验证 |
| 3 | 链接提取 | 正则匹配 `sfsdw//r/` 后的 ID | 提取成功 | 待验证 |
| 4 | 浏览器启动 | `openclaw browser start` 执行 | 执行 | 待验证 |
| 5 | 错误处理 | 链接无效时微信通知用户 | 通知 | 待验证 |

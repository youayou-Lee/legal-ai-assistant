### 修复记录

**问题**：SKILL.md 的 YAML frontmatter 缺少闭合 `---`。

**修复前**：
```yaml
---
name: court-document-handler
description: Use when message contains "广西法院短信平台"...

# Court Document Handler    ← 被当成YAML解析，失败
```

**修复后**：
```yaml
---
name: court-document-handler
description: Use when message contains "广西法院短信平台"...
metadata:
  { "openclaw": { "emoji": "⚖️", "os": ["linux", "darwin", "win32"] } }
---

# Court Document Handler    ← 正确识别为Markdown正文
```

**修复效果**：
- 修复前：`openclaw skills list` 不显示该 Skill，收到短信时 AI 默认用 WebSearch 回复
- 修复后：`openclaw skills list` 显示 `✓ ready ⚖️ court-document-handler`，Skill 可被正常触发

---

## 关键发现

1. **OpenClaw Skill 加载失败是静默的**：YAML frontmatter 格式错误不会报错，Skill 直接不加载，AI 收到匹配消息时找不到 Skill，默认用 WebSearch 回复。表现是"AI 不用 Skill"，根因是"Skill 没被加载"。
2. **法院系统必须用浏览器**：法院电子送达平台需要验证码交互，`web_fetch` 无法处理，必须用 `openclaw browser` 系列命令。

## 仍存在的问题

- 完整端到端流程未经真实数据验证（需要有效法院短信链接）
- 验证码、微信账号硬编码在 Skill 中
- 仅支持广西法院短信平台

# 证据目录生成

## 技术方案
OpenClaw Skill（`SKILL.md` + 8步强制推理框架）

## 一句话说明
根据案件材料按强制推理步骤生成符合法院要求的证据目录（一审原告），区分证明内容与证明目的，自动检测证据链缺口和金额不一致。

## 当前状态
v1已完成模拟测试，待真实案例律师验证

## 部署位置
`~/.agents/skills/evidence-directory/`

## 法律依据

《最高人民法院关于民事诉讼证据的若干规定》（2019年修正）第十九条第一款：
> 当事人应当对其提交的证据材料逐一分类编号，对证据材料的来源、证明对象和内容作简要说明，签名盖章，注明提交日期，并依照对方当事人人数提出副本。

## 核心设计

### 证明内容 vs 证明目的（核心区分）

| 维度 | 证明内容 | 证明目的 |
|------|---------|---------|
| 性质 | 客观事实描述 | 法律论证目标 |
| 示例 | "被告于2023年3月15日出具借条，载明借款100,000元" | "证明原被告之间存在合法有效的民间借贷法律关系" |
| 要求 | 如实描述，不加入主观判断 | 具体指向法律事实，禁止空话 |

### 简单/复杂案件自适应

| 类型 | 判断标准 | 输出格式 |
|------|---------|---------|
| 简单案件 | 证据≤5项，可归入同一场景 | 平铺列表，不分组 |
| 复杂案件 | 证据>5项，或涉及多个待证事实 | 按待证事实分组 |

⚠️ 无论简单或复杂，证据链完整性检查和金额一致性校验都必须执行。

## 8步强制推理

| Step | 内容 | 核心产出 |
|------|------|---------|
| 1 | 提取证据条目 | 证据清单 + 可能缺失的证据提示 |
| 2 | 确定案由和待证事实 | 待证事实列表 + 复杂度判断 |
| 3 | 证据编排 | 平铺/分组 + 缺失证据标注 |
| 4 | 撰写证明内容和证明目的 | 每份证据的证明内容和证明目的 |
| 5 | 证据链完整性检查 | 缺口分析表 + 金额一致性校验 |
| 6 | 自检 | 8项质量标准逐项检查 |
| 7 | 输出证据目录 | 标准格式表格 |
| 8 | 输出docx | 转换为Word文档 |

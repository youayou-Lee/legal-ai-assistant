# 起诉状法定格式与律师需求研究

## 起诉状法定格式

### 法律依据

《民事诉讼法》第121条规定，起诉状应当记明下列事项：
1. 原告的姓名、性别、年龄、民族、职业、工作单位、住所、联系方式，法人或者其他组织的名称、住所和法定代表人或者主要负责人的姓名、职务、联系方式
2. 被告的姓名、性别、工作单位、住所等信息，法人或者其他组织的名称、住所等信息
3. 诉讼请求和所根据的事实与理由
4. 证据和证据来源，证人姓名和住所

### 2024年最高法示范文本更新

2024年3月，最高人民法院发布《民事起诉状（公民起诉用）示范文本》，更新要点：
- 明确要求诉讼请求"具体、明确"
- 强调事实与理由"简明扼要，条理清晰"
- 要求列明证据清单（证据名称、证据来源、证明目的）
- 规范化排版（段落分明、层次清晰）

## 质量标准

### 基本质量维度
1. **诉讼请求明确具体**
   - 金额精确到分
   - 请求项无遗漏
   - 请求项之间无矛盾

2. **事实陈述清晰**
   - 时间线完整
   - 因果关系明确
   - 关键事实不遗漏

3. **法律关系准确**
   - 案由认定正确
   - 当事人适格
   - 法律适用准确

4. **管辖法院正确**
   - 被告住所地
   - 合同履行地
   - 专属管辖（如适用）

5. **证据链完整**
   - 证据与事实对应
   - 证据形式合法
   - 证明目的清晰

6. **法言法语规范**
   - 使用法律术语
   - 避免口语化
   - 引用法条准确

## 律师需求调研

### 吴律的审核需求

根据 **项目记忆：吴律反馈**，吴律认为起诉状审核应包括：

#### 必要审核项
1. **当事人适格**
   - 原告是否有诉讼主体资格
   - 被告是否明确
   - 是否有遗漏的必要共同诉讼人

2. **诉讼请求是否遗漏**
   - 是否遗漏了应主张的权利
   - 金额计算是否准确
   - 请求项之间是否有冲突

3. **事实理由完整准确**
   - 事实是否全面
   - 时间线是否清晰
   - 法律关系认定是否准确

4. **法律适用准确**
   - 法条引用是否正确
   - 法律关系认定是否准确
   - 是否有适用法律错误

5. **证据链完整**
   - 证据是否充分
   - 证据与事实是否对应
   - 是否有证据形式瑕疵

#### 吴律认为AI有必要的功能
- 文书及证据目录生成
- 风险分析（胜诉率、证据薄弱点）
- 法条匹配（自动查找相关法条）
- 庭审提纲（争议焦点、举证质证要点）

#### 吴律认为无必要的功能
- 过多展示过程性东西（律师不需要看AI的推理过程）
- 简单问题复杂化（不要为了"智能"而增加复杂度）

## 当前Skill的设计考虑

### 符合法定格式
- ✓ 严格按《民诉法》第121条结构
- ✓ 诉讼请求列明为分项
- ✓ 事实理由分段陈述
- ✓ 证据清单独立列明

### 满足质量标准
- ✓ Step 8质量评估（5个维度）
- ✓ 强制要求诉讼请求明确具体
- ✓ 强制要求事实时间线完整
- ✓ 强制要求证据清单

### 回应律师需求
- ✓ Step 3当事人适格性检查
- ✓ Step 5c列明诉讼请求（防止遗漏）
- ✓ Step 7事实理由生成（完整准确）
- ✓ Step 4a + 4b法条引用（法律适用）
- ✓ Step 6证据清单整理（证据链）

### 改进空间
- 风险分析尚未实现（胜诉率评估）
- 庭审提纲尚未生成（争议焦点、举证质证要点）
- 过程性输出可能过多（Step 5a、5b的计算过程可以简化）

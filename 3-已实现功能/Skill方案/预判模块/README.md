# 对方策略预判模块

## 技术方案
OpenClaw Skill（`SKILL.md` + 6步强制换位思考框架）

## 一句话说明
站在对方律师视角，系统推演对方可能采取的答辩策略、质证意见和辩论重点。

## 部署位置
`~/.agents/skills/opponent-prediction/`

## 核心流程（6步）

0. **锁定用户/对方身份** — 必须先确认原告还是被告
1. **从对方视角重构事实** — 全部站在对方立场
2. **预判答辩点（5维度）** — 诉讼主体+管辖+时效+事实+法律
3. **预判质证意见** — 按证据三性（真实性、合法性、关联性）攻击
4. **预判辩论重点** — 争议焦点+论证逻辑
5. **盲区扫描+应对建议** — 强制找自己方案的弱点

## 核心设计洞察

AI 天然倾向站在用户视角分析，必须通过 Step 0 身份锁定和 Step 5 盲区扫描来约束。

## v1 测试结果（模拟民间借贷案例）

全部验证点命中：
- 答辩点预判 5/5：砍头息抗辩、已还款抗辩、诉讼时效、利息基数、LPR上限
- 质证意见 3/3：借款协议、转账凭证、微信聊天记录
- 辩论重点 3/3：本金认定、还款性质、利息起算点
- 盲区发现 5个（2高风险 + 3中等风险）

## 概率评估说明

每个预判点标注"高/中/低"概率——这是**策略出现的可能性**，不是胜率预测。opponent-prediction 遵循全局禁止概率输出的规则，"高/中/低"属于允许范围。

## 📋 一键测试

```
帮我预判对方律师会怎么打这个案子：

【我的身份】原告方（出借人）

【案件事实】
我是原告，2024年6月1日借给被告150,000元，年利率6%，借期6个月（2024年12月1日到期）。有借款协议。

特殊事实：
1. 借款时我让被告预先支付了9,000元"利息"（从转账中直接扣除）
2. 我实际转账给被告141,000元（150,000 - 9,000）
3. 被告2024年10月1日还款20,000元，没说还本金还是利息
4. 2024年12月15日我微信催款，被告回复"过完年再还"

【我的诉讼请求】
1. 判令被告偿还借款本金150,000元
2. 判令被告支付利息（按年利率6%计算）
3. 诉讼费用由被告承担

【我方的证据】
1. 借款协议：2024年6月1日签署，载明借款金额150,000元，年利率6%
2. 银行转账记录：2024年6月1日转账141,000元至被告账户
3. 微信聊天记录：2024年12月15日催款，被告回复"过完年再还"
4. 被告还款记录：2024年10月1日收到被告转账20,000元
```

**验证重点**：
1. 身份识别：应自动识别"我=原告方"
2. 砍头息预判：应预判被告主张实际本金为141,000元
3. 已还款预判：应预判被告主张已还20,000元应扣除
4. 利息基数预判：应预判被告主张利息以141,000元为基数
5. 质证攻击：应攻击转账金额与协议金额不一致
6. 盲区发现：应发现砍头息风险、还款抵充争议、利息起算点争议
7. 全程视角：不应出现为原告辩护的语言，必须站在被告律师立场

## 测试目标

验证预判模块能否准确预判对方律师的答辩策略、质证意见和辩论重点，特别是能否发现用户自身案件的盲区。

## 测试案例：民间借贷纠纷

### 基本案情

**案件类型**：民间借贷纠纷

**用户身份**：原告方

**案件事实**：

1. **借款基本信息**
   - 借款金额：150,000元
   - 年利率：6%
   - 借期：6个月
   - 借款日期：2024年6月1日
   - 到期日期：2024年12月1日

2. **特殊事实（风险点）**
   - **预付利息**：借款时被告预先支付了9,000元"利息"（砍头息嫌疑）
   - **实际转账**：原告实际转账141,000元（150,000 - 9,000）
   - **部分还款**：被告已还款20,000元（未注明是本金还是利息）
   - **催款记录**：被告微信回复"过完年再还"

3. **原告主张**
   - 请求判令被告偿还借款本金150,000元
   - 请求判令被告支付利息（按年利率6%计算）
   - 诉讼费用由被告承担

### 证据清单

1. **借款协议**：2024年6月1日签署，载明借款金额150,000元，年利率6%
2. **银行转账记录**：2024年6月1日转账141,000元至被告账户
3. **微信聊天记录**：2024年12月15日催款，被告回复"过完年再还"
4. **被告还款记录**：2024年10月1日收到被告转账20,000元

### 测试输入

将以上案情和证据作为输入，要求预判模块分析对方（被告）可能采取的策略。

## 验证点清单

### 1. 答辩点预判（5项）

#### 1.1 程序性答辩

- [ ] **管辖权异议**：预判是否会提出（低概率，本案无管辖问题）
- [ ] **诉讼时效抗辩**：预判是否会提出（低概率，未过时效）

#### 1.2 实质性答辩

- [ ] **砍头息抗辩**：预判是否会主张实际借款为本金141,000元（高风险）
- [ ] **已还款抗辩**：预判是否会主张已还20,000元应予扣除（高风险）
- [ ] **利息基数抗辩**：预判是否会主张利息应以141,000元为基数（高风险）
- [ ] **利息起算点抗辩**：预判是否会主张利息从到期日起算而非催款日起算
- [ ] **LPR上限抗辩**：预判是否会主张6%年利率超过LPR四倍（需验证是否确实超过）

### 2. 质证意见预判（3项）

- [ ] **借款协议质证**：预判是否会攻击协议真实性、合法性或关联性
- [ ] **转账凭证质证**：预判是否会指出转账金额与协议金额不一致
- [ ] **微信聊天记录质证**：预判是否会质疑聊天记录的真实性或证明力

### 3. 辩论重点预判（3项）

- [ ] **本金认定争议**：150,000元 vs 141,000元
- [ ] **还款性质争议**：20,000元是还本还是还息
- [ ] **利息计算争议**：基数、起算点、上限

### 4. 盲区发现

- [ ] **高风险盲区**：砍头息问题，可能导致本金核减
- [ ] **高风险盲区**：还款抵充顺序争议，可能导致利息计算错误
- [ ] **中等风险盲区**：利息起算点可能不是催款日而是到期日
- [ ] **中等风险盲区**：证据链可能不完整（缺少催款记录的原始载体）
- [ ] **中等风险盲区**：被告可能提出已过诉讼时效（如能证明最后一次催款时间）

## 测试方法

### Step 1: 输入测试案例

将上述案情和证据输入预判模块，触发6步推理流程。

### Step 2: 验证身份识别

检查 Step 0 是否正确识别：
- 用户身份：原告方
- 对方身份：被告方
- 案件类型：民间借贷纠纷

### Step 3: 验证推理完整性

检查是否完成所有6个步骤：
- [ ] Step 0: 身份锁定
- [ ] Step 1: 对方视角事实重构
- [ ] Step 2: 答辩点预判
- [ ] Step 3: 质证意见预判
- [ ] Step 4: 辩论重点预判
- [ ] Step 5: 盲区扫描与应对建议
- [ ] Step 6: 质量自检

### Step 4: 验证预判准确性

对照验证点清单，逐项检查：

1. **答辩点预判**：是否命中5个关键答辩点
2. **质证意见预判**：是否对3份证据都做了预判
3. **辩论重点预判**：是否识别出3个核心争议点
4. **盲区发现**：是否发现至少5个风险点

### Step 5: 验证换位思考质量

检查是否真正站在被告视角：
- [ ] 事实重构是否从被告角度重新审视
- [ ] 答辩点是否覆盖被告可能的所有主张
- [ ] 质证意见是否具有攻击性（而非为原告辩护）
- [ ] 辩论重点是否体现被告的核心利益

### Step 6: 验证应对建议可操作性

检查 Step 5 的应对建议是否：
- [ ] 针对每个高风险盲区给出具体应对方案
- [ ] 建议是否可操作（不是"注意XX"而是"准备XX"）
- [ ] 是否列出需要补充的证据或材料

## 测试案例设计思考

### 为什么选择民间借贷纠纷？

1. **高频案件**：民间借贷是基层法院最常见的案件类型之一
2. **争议点典型**：砍头息、还款抵充、利息计算等争议点具有代表性
3. **风险点明确**：容易设计高风险盲区，验证模块的盲区发现能力

### 为什么设计这些风险点？

1. **砍头息（9,000元预付利息）**：民间借贷中最常见的高风险争议点，直接影响本金认定
2. **已还款未注明性质（20,000元）**：引发还款抵充顺序争议，影响利息计算
3. **微信催款记录**：证据效力较弱，容易被质证攻击
4. **利息起算点模糊**：到期日 vs 催款日 vs 催款日，容易产生争议

这些风险点都是真实案件中常见的问题，能够有效验证预判模块的实用性。

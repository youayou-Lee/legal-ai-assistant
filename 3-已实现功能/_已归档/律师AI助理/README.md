# 律师AI助理

## 基本信息

- **技术方案**: OpenClaw部署的专业法律领域AI助手
- **一句话说明**: 智能识别法律任务，执行标准化法律工作流，自动生成结构化总结报告
- **部署位置**: `~/.agents/skills/legal-assistant/`
- **当前状态**: 已开发，待真实案例验证

## 功能描述

律师AI助理是一个专门为法律领域设计的智能技能，能够：

### 核心功能

1. **法律领域任务识别**
   - 自动识别输入内容是否属于法律任务
   - 区分民事、刑事、行政等不同法律领域
   - 判断任务复杂度并匹配相应处理策略

2. **标准化法律工作流**
   - **明确问题**: 提炼核心法律争议点
   - **搜索依据**: 检索相关法律法规、司法解释
   - **适用分析**: 分析法律规定的具体适用情形
   - **识别风险**: 潜在法律风险点识别
   - **建议方案**: 提供结构化解决方案建议

3. **自动总结记录**
   - 生成结构化法律分析总结
   - 记录关键法律要点
   - 整理法条引用和判例参考
   - 自动保存至 `memory/legal-summaries.md`

### 禁止项（已加入概率禁止条款）

- **禁止输出概率/预判数字**: 胜率百分比、支持率、置信度、量刑百分比、案件结果概率等
- **禁止预判性结论**: 如"大概率胜诉"、"可能判X年"、"胜算较高"等
- **禁止代替专业法律意见**: 明确提示用户需咨询专业律师

## 实现架构

### 技术架构

```
律师AI助理
├── 任务识别层
│   ├── 法律领域分类器
│   ├── 任务类型判断
│   └── 复杂度评估
├── 工作流执行层
│   ├── 问题明确模块
│   ├── 法律检索模块
│   ├── 适用分析模块
│   ├── 风险识别模块
│   └── 方案生成模块
├── 总结输出层
│   ├── 结构化总结生成
│   ├── 法条引用整理
│   ├── 风险提示生成
│   └── 记录自动保存
└── 质量控制层
    ├── 禁止项检查
    ├── 逻辑验证
    └── 输出格式化
```

### 工作流程

1. **输入接收**: 接收用户提出的法律问题或任务
2. **任务识别**: 判断是否为法律任务及类型
3. **工作流执行**: 按标准化流程进行处理
4. **风险检查**: 确保输出不包含禁止项
5. **总结生成**: 生成结构化分析报告
6. **记录保存**: 自动保存至指定memory文件

## 与其他功能的联动

### 案件管理联动
- 与案件管理系统集成，自动关联案件编号
- 读取案件历史记录，提供连贯性分析
- 更新案件状态和处理进度

### 法律文书联动
- 为法律文书生成模块提供基础分析
- 自动提取关键信息用于文书起草
- 确保文书内容的法律准确性

### 风险评估联动
- 为案件风险评估提供基础数据
- 识别潜在风险点，触发相应预警
- 与辩护工作流协同，优化辩护策略

## 应用场景

1. **法律咨询**: 为初级律师提供法律分析参考
2. **案件初筛**: 快速判断案件类型和处理方向
3. **法律研究**: 辅助法律法规检索和应用
4. **风险预警**: 识别案件潜在风险点
5. **知识管理**: 结构化整理法律分析经验

## 技术特点

- **专业化**: 针对法律领域深度优化
- **标准化**: 遵循法律工作流程规范
- **结构化**: 输出格式统一规范
- **可追溯**: 分析过程完整记录
- **合规性**: 严格遵守禁止项要求

> **注意**: 本技能为辅助工具，不能替代专业律师的法律意见。所有分析结果需经专业律师审核确认。

# 案件分析与报告质保 Skill方案

## 基本信息

**技术方案**：基于 Claude Code 技能生态的刑事案件智能分析流水线
**一句话说明**：提供从案件分析到报告质保的全流程自动化支持
**部署位置**：`~/.claude/skills/`
**当前状态**：已开发，待真实案例验证

## 功能描述

本方案包含4个协同工作的技能，形成完整的案件分析与报告质保流水线：

1. **案件分析**（case-analyzer）：从知识图谱中提取洞察、生成交互式分析报告
2. **质量保证**（case-qa）：确保报告内容有据可依，实现增量验证
3. **案件审查**（case-review）：审查分析报告质量，验证证据可溯源性
4. **文书生成**（case-doc-draft）：基于分析报告生成专业法律文书

## 实现架构

### 技能调用关系图

```
案件材料
    ↓
case-preprocess (预处理)
    ↓
case-graph-build (构建知识图谱)
    ↓
case-analyzer ←────────────┐
    ↓                 ↓
case-doc-draft ←─ case-qa (质量验证)
    ↓
case-review (最终审查)
```

### 核心功能模块

#### 1. case-analyzer（案件分析模块）
- **部署路径**：`~/.claude/skills/case-analyzer/`
- **核心功能**：
  - 时间线分析：自动梳理案件时间节点
  - 证据链组织：构建证据间的逻辑关系
  - 争议焦点提取：识别案件关键争议点
  - 洞察发现：发现矛盾点、证据缺口、风险和机会
  - 交互问答：基于分析结果提供智能问答
- **依赖关系**：依赖 case-graph-build 和 case-preprocess
- **命令**：`/case-analyze <案件名>`

#### 2. case-qa（案件分析报告质量保证）
- **部署路径**：`~/.claude/skills/case-qa/`
- **核心功能**：
  - 增量验证：基于hash实现内容变更检测
  - 关键断言提取与验证：识别报告中的关键主张
  - 图谱辅助验证：利用知识图谱验证分析结论
- **验证目标**：验证 defense-analysis 的输出质量
- **命令**：`/case-qa <报告路径> <案件目录>`

#### 3. case-review（案件审查）
- **部署路径**：`~/.claude/skills/case-review/`
- **核心功能**：
  - 证据可溯源性验证：确保每项结论都有证据支持
  - 逻辑推理审查：分析推理过程的严密性
  - 法律依据验证：检查引用法律条款的准确性
  - 完整性检查：确认分析覆盖所有重要方面
- **重要约束**：包含概率/预判禁止条款，专注于报告质量而非案件结果预判
- **命令**：`/case-review <分析报告路径>`

#### 4. case-doc-draft（法律文书生成）
- **部署路径**：`~/.claude/skills/case-doc-draft/`
- **核心功能**：
  - 辩护词生成：基于分析报告生成辩护意见
  - 取保候审申请：撰写取保候审申请书
  - 法律意见书：生成专业法律分析意见
  - 质证意见：针对关键证据的质证要点
- **依赖关系**：依赖 case-manager 和 case-analyzer
- **适用场景**：主要面向刑事案件
- **命令**：`/case-doc-draft <案件名> <文书类型>`

## 与其他功能的联动

### 上游联动
- **case-preprocess**：提供预处理后的标准化案件材料
- **case-graph-build**：构建案件知识图谱，提供分析基础
- **defense-legal**：提供法律适用分析，支持案件定性

### 下游联动
- **defense-brief**：基于分析报告生成辩护意见书
- **defense-cross-exam**：利用分析结果生成质证意见
- **defense-sentencing**：结合分析进行量刑预测

### 并行协作
- **case-manager**：管理案件状态和进度跟踪
- **service-log**：记录服务过程和质量反馈
- **defense-workflow**：协调整个辩护工作流程

## 技术特色

1. **流水线设计**：四个技能形成完整的分析-验证-审查-生成流水线
2. **增量验证**：通过hash实现报告内容的增量验证机制
3. **图谱支持**：基于知识图谱进行深度分析和验证
4. **质量导向**：强调报告质量，避免结果预判
5. **专业适配**：针对刑事案件场景深度优化

# 案件分析与报告质保测试文档

## 📋一键测试区域

```bash
# 1. 创建测试案件
/case new "测试案件"

# 2. 预处理案件材料
/case-preprocess "测试案件" --source "data/test-materials/"

# 3. 构建知识图谱
/case-graph-build "测试案件"

# 4. 执行案件分析
/case-analyze "测试案件"

# 5. 质量验证
/case-qa "reports/test-case-analysis.md" "测试案件"

# 6. 案件审查
/case-review "reports/test-case-analysis.md"

# 7. 生成法律文书
/case-doc-draft "测试案件" --type "defense-brief"
```

## 测试案例背景

### 案例名称：张某盗窃案测试案例

### 案情概述
张某因涉嫌盗窃罪被公安机关刑事拘留。案卷包含以下材料：
- 犯罪嫌疑人供述材料3份
- 证人证言5份
- 物证照片及鉴定报告
- 书证（银行流水、购物小票）
- 鉴定意见书（价格鉴定）
- 视听资料（监控录像截图）

### 测试环境
- Claude Code 版本：v2.0
- Skill 部署路径：`~/.claude/skills/`
- 测试数据目录：`data/test-materials/`
- 输出报告目录：`reports/`

## 验证点清单

### 1. case-analyzer 验证点

#### 时间线分析
- [ ] 自动识别并排序所有时间节点
- [ ] 时间线格式清晰，包含具体日期和时间
- [ ] 时间间隔标注准确，识别关键时间窗口
- [ ] 能发现时间线上的矛盾点

#### 证据链组织
- [ ] 证据分类正确（人证、物证、书证、鉴定意见等）
- [ ] 证据间关系图谱构建完整
- [ ] 证据链逻辑严密，支持分析结论
- [ ] 证据与指控的关联性分析准确

#### 争议焦点提取
- [ ] 识别至少3个主要争议焦点
- [ ] 每个焦点都有证据支持和分析
- [ ] 焦点间关系梳理清晰
- [ ] 包含控辩双方的不同观点

#### 洞察发现
- [ ] 发现证据矛盾（至少1处）
- [ ] 识别证据缺口（至少1处）
- [ ] 指出程序风险（至少1项）
- [ ] 辩护机会识别（至少1个）

### 2. case-qa 验证点

#### 增量验证
- [ ] 首次运行生成完整验证报告
- [ ] 修改报告后增量检测变更内容
- [ ] hash值计算准确，变更检测精确
- [ ] 验证日志记录完整

#### 断言提取
- [ ] 自动提取10+关键断言
- [ ] 断言分类准确（事实、法律、推理）
- [ ] 每个断言都有对应证据支持
- [ ] 断言间逻辑关系清晰

#### 图谱验证
- [ ] 利用知识图谱验证80%以上断言
- [ ] 无法验证的断言标注清楚
- [ ] 图谱验证结果与原文一致
- [ ] 验证过程可追溯

### 3. case-review 验证点

#### 证据可溯源性
- [ ] 每项结论都有对应证据引用
- [ ] 证据引用格式规范（证据名称+页码）
- [ ] 证据链完整无断裂
- [ ] 证据真实性评估准确

#### 逻辑推理审查
- [ ] 推理过程符合逻辑规则
- [ ] 无逻辑跳跃和循环论证
- [ | 假设条件明确且合理
- [ ] 结论与前提逻辑一致

#### 法律依据验证
- [ ] 引用法律条文准确
- [ ] 法律适用正确
- [ ] 先例引用恰当
- [ ] 法律更新已同步

#### 完整性检查
- [ ] 涵盖所有关键指控
- [ ] 分析了所有重要证据
- [ ] 考虑了所有可能的辩护点
- [ ] 无遗漏的重要事项

### 4. case-doc-draft 验证点

#### 辩护词生成
- [ ] 格式符合司法文书规范
- [ ] 事实陈述客观准确
- [ ] 法律论证严密
- [ ] 辩护策略明确合理

#### 取保候审申请
- [ ] 符合《刑事诉讼法》规定
- [ ] 社危险性分析充分
- [ ] 保证条件明确
- [ ] 申请理由充分

#### 法律意见书
- [ ] 结构完整（案情、分析、意见）
- [ | 法律依据引用准确
- [ | 结论明确有据
- [ | 语言专业规范

#### 质证意见
- [ ] 针对每份关键证据
- [ ] 三性分析（真实性、合法性、关联性）
- [ | 质证意见具体可操作
- [ ] 组成系统质证方案

## 评分标准

### 性能评分（40分）
- 响应时间：≤30秒（10分）
- 内存占用：≤2GB（10分）
- 并发处理：支持3个同时案件（10分）
- 稳定性：连续运行4小时无崩溃（10分）

### 准确性评分（40分）
- 事实准确度：≥95%（10分）
- 法律适用准确：≥90%（10分）
- 逻辑严密性：≥90%（10分）
- 完整性：≥90%（10分）

### 易用性评分（20分）
- 命令友好度：命令简单易记（5分）
- 错误提示：清晰明确（5分）
- 输出格式：符合阅读习惯（5分）
- 帮助文档：完整易懂（5分）

### 总分计算
- 优秀：90-100分
- 良好：80-89分
- 合格：70-79分
- 需改进：<70分

## 测试数据准备

### 输入数据规范
1. **案件材料**：
   - PDF格式，扫描件清晰
   - 包含完整的卷宗封面
   - 目录结构清晰

2. **测试标准**：
   - 真实案例脱敏处理
   - 包含常见争议类型
   - 覆盖各类证据形式

### 预期输出标准
1. **分析报告**：
   - 结构化输出（Markdown格式）
   - 包含可交互的问答功能
   - 支持图谱可视化

2. **验证报告**：
   - 包含详细的验证过程
   - 标注问题等级和修复建议
   - 支持导出验证清单

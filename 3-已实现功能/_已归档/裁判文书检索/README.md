# 裁判文书检索 Skill方案

## 基本信息

**技术方案**：基于 Playwright 的裁判文书爬取和案例检索系统
**一句话说明**：从中国裁判文书网爬取法律文书并进行智能案例检索分析
**部署位置**：`~/.claude/skills/wenshu-scraper/` 和 `~/.claude/skills/case-precedent-search/`
**当前状态**：已开发，战略方向已废弃（2026-04-04），作为技术储备保留

## 功能描述

本方案包含两个协同工作的技能，提供从文书爬取到案例检索的完整解决方案。主要面向刑事案件的案例检索需求，为律师提供相似案例参考。

**重要说明**：由于律师更倾向于使用威科先行/Alpha等付费产品，且我们没有数据库积累，项目战略方向已确认"类案检索整体方向废弃"。本方案作为技术储备保留，但不作为产品重点。

## 实现架构

```
中国裁判文书网
    ↓
```

### 核心功能模块

#### 1. wenshu-scraper（裁判文书爬虫）
- **部署路径**：`~/.claude/skills/wenshu-scraper/`
- **核心功能**：
  - 登录与会话管理（full storage_state 持久化）
  - 反检测措施（模拟真实用户行为）
  - SPA 结构处理（动态加载内容）
  - 结果过滤与 URL 解析
  - 内容提取（正则模式）
  - 相关性评分
- **技术特点**：
  - 使用 Playwright 实现自动化
  - 支持会话持久化，避免重复登录
  - 智能反爬虫策略
  - 高效的内容提取
- **适用场景**：需要批量获取裁判文书时使用

#### 2. case-precedent-search（案例检索）
- **部署路径**：`~/.claude/skills/case-precedent-search/`
- **核心功能**：
  - 基于相似因素的案例识别
  - 判决结果分析
  - 参考价值评估
  - 量刑分布分析
  - 模式总结
- **技术特点**：
  - 多维度相似度计算
  - 深度案例分析
  - 量化评估体系
  - 可视化展示
- **适用场景**：为律师提供类似案例参考
- **使用限制**：主要用于技术演示，实际业务中推荐使用付费数据库

## 技术特色

### 1. wenshu-scraper 技术特色

#### 会话管理创新
- **持久化存储**：完整保存登录状态，避免重复验证
- **智能刷新**：自动检测会话过期并重新登录
- **多账号轮询**：支持多个账号轮换使用，提高稳定性

#### 反检测策略
- **行为模拟**：模拟真实用户浏览行为
- **随机延迟**：加入随机操作间隔
- **IP 轮换**：支持代理 IP 轮换
- **设备指纹**：模拟真实设备特征

#### SPA 页面处理
- **动态等待**：智能等待内容加载完成
- **滚动加载**：自动处理无限滚动加载
- **异步处理**：高效处理异步加载的内容

### 2. case-precedent-search 技术特色

#### 多维度相似度计算
- **案情相似度**：指控事实、犯罪情节等关键要素比对
- **法律适用相似度**：罪名、法律条款适用情况
- **情节相似度**：从轻从重情节匹配
- **判决结果相似度**：刑罚类型和幅度对比

#### 深度分析能力
- **趋势分析**：历年判决趋势变化
- **地域差异**：不同地区的判决习惯
- **法官风格**：不同法官的判决特点
- **类型分析**：不同案件类型的判决模式

#### 量化评估体系
- **相似度得分**：0-100 分的量化评分
- **参考价值指数**：综合评估参考价值
- **匹配度分析**：多维度匹配结果展示
- **风险提示**：潜在差异和风险点提示

## 输出报告格式

### 1. 案例检索报告
- **案例列表**：按相似度排序的案例列表
- **案情对比**：与目标案情的对比分析
- **判决结果**：相似案例的判决结果统计
- **参考价值**：案例参考价值评估

### 2. 量刑分析报告
- **量刑分布**：相似案例的刑罚分布统计
- **影响因素**：影响量刑的关键因素分析
- **趋势分析**：近年来的量刑趋势
- **建议范围**：基于案例的量刑建议

### 3. 模式总结报告
- **地域差异**：不同地区的判决习惯
- **时间变化**：判决标准的历史变化
- **类型特征**：不同类型案件的特点
- **审判规律**：案件审理的一般规律

## 使用方法

### wenshu-scraper 使用
```bash
# 爬取指定案例
/wenshu-scraper <案例ID> --output <保存路径>

# 批量爬取案例
/wenshu-scraper <案例列表文件> --batch --output <目录>

# 自定义搜索爬取
/wenshu-scraper <搜索关键词> --pages <页数> --output <目录>
```

### case-precedent-search 使用
```bash
# 检索相似案例
/case-precedent-search <案件目录> --output <报告路径>

# 分析特定罪名
/case-precedent-search <案件目录> --offense <罪名> --output <报告路径>

# 生成量刑分析
/case-precedent-search <案件目录> --analysis sentencing --output <报告路径>
```

## 技术限制

### 1. 数据时效性
- **限制**：爬取的数据可能存在延迟
- **原因**：裁判文书网更新频率限制
- **应对**：建议结合其他权威数据库使用

### 2. 数据覆盖范围
- **限制**：无法覆盖所有裁判文书
- **原因**：网站反爬虫机制限制
- **应对**：重点关注重要和典型案例

### 3. 法律效力
- **限制**：爬取数据仅供参考
- **原因**：官方数据库具有权威性
- **应对**：重要案件需核对官方来源

### 4. 使用风险
- **限制**：可能违反网站使用条款
- **原因**：大规模爬取可能被禁止
- **应对**：合理控制访问频率，遵守robots协议

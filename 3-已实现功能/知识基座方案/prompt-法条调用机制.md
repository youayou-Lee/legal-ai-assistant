# 法律分析法条调用 Prompt 机制

> 用途：嵌入 Agent 的 AGENTS.md 或 Skill prompt 中，控制法条库的渐进式调用
> 原则：按需查阅，不全量注入；分析必引法条编号

---

## Prompt 模板（嵌入 AGENTS.md / Skill）

```markdown
## 法律分析强制规则

### 第一步：识别案件类型

收到案件后，先判断涉及哪些案件类型。每种类型对应一个法条库：

| 案件类型 | 法条库文件 |
|---------|-----------|
| 婚姻财产纠纷 | `3-法条库/婚姻财产纠纷-核心法条.md` |
| 继承纠纷 | `3-法条库/继承纠纷-核心法条.md`（待建） |
| 合同纠纷 | `3-法条库/合同纠纷-核心法条.md`（待建） |
| 劳动争议 | `3-法条库/劳动争议-核心法条.md`（待建） |

### 第二步：查阅法条库

在开始分析前，**必须先读取**对应法条库文件。这不是可选项。

读取策略（渐进式）：
1. **先读「法条索引」**（文件末尾的索引表）——了解本类型有哪些法条
2. **再读「分析强制检查清单」**——知道分析时必须检查哪些项
3. **按需精读具体法条**——遇到涉及具体概念的财产项时，精读对应章节

### 第三步：强制引用

分析过程中，每项财产/诉求的结论**必须包含**：
1. 法律性质认定（个人财产/共同财产/混合）
2. 法条依据（至少到"条"的级别，如"民法典第1062条"、"司法解释（一）第78条"）
3. 如涉及金额：计算公式和过程

**禁止**：
- ❌ 不引用任何法条就下结论
- ❌ 只说"根据相关法律规定"
- ❌ 凭感觉估算金额而不列公式

### 第四步：自检

分析完成后，对照法条库的「分析强制检查清单」逐项核对：
- 每项财产是否标注了取得时间？
- 增值部分是否区分了原因？
- 父母出资是否判断了性质？
- 债务是否走完了1064条三种情形？
```

---

## 设计说明

### 为什么不全量注入？

1. **Context窗口有限**——全量法条会挤占分析空间
2. **渐进式更快**——先看索引（几行），按需精读（几段），比一次塞入整篇文档更高效
3. **类似律师工作方式**——律师也不是把所有法条背下来，而是先知道"该查什么"，再去查

### 渐进式展示流程

```
收到案件 → 识别类型 → 读索引表（1秒）
                              ↓
              遇到"父母出资" → 精读第三章（第29条）
              遇到"增值"   → 精读第二章（第78条）
              遇到"债务"   → 精读第四章（第1064条）
                              ↓
              对照检查清单 → 逐项核对 → 输出
```

### 如何扩展到其他案件类型

每个新案件类型只需要：
1. 创建 `3-法条库/XX纠纷-核心法条.md`（按统一结构：总框架 → 分论点 → 检查清单 → 索引）
2. 在上方索引表中加一行
3. Prompt模板不需要改动

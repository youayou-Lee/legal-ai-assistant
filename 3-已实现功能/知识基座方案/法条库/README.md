# 法条库 - 法律知识基座

## 功能概述

**法条库**是一个结构化的法律知识资源系统，采用渐进式调用机制为AI助手提供精准的法律条文支持。它既不是Skill也不是Agent，而是被Skill/Agent调用的底层知识基座。

## 技术方案

### 核心机制：结构化Markdown + 渐进式调用

```
┌─────────────┐
│  案件输入   │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  识别案件类型 │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  读取法条索引 │  ← 轻量级（仅索引）
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  按需精读章节 │  ← 仅读取相关法条
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  对照检查清单 │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  输出分析结果 │
└─────────────┘
```

### 为什么不全量注入法条？

1. **Context窗口限制**：全量注入会占用大量Token，影响AI推理质量
2. **效率考量**：渐进式调用更快，类似律师查资料的工作方式
3. **精准性**：按需读取避免无关信息干扰

## 核心设计特点

### 1. 渐进式调用机制
- **第一步**：读取索引表，了解法条库覆盖范围
- **第二步**：遇到具体法律概念时，精读对应章节
- **第三步**：对照检查清单，逐项验证是否已覆盖

### 2. 结构化Markdown格式
- 每个法条包含：条文编号、条文内容、适用场景、司法解释要点
- 使用Obsidian标签系统便于检索和维护
- 支持wikilink跳转，建立法条间的关联

### 3. 强制引用规则
- AI分析时必须引用法条到"条"级别（如"民法典第1064条"）
- 不得仅凭常识推断，必须有法条依据
- 法条引用必须与案件事实一一对应

### 核心发现
- **婚姻财产领域**：AI缺的是**知识**（法条），法条库是关键解法
- **合同纠纷领域**：AI缺的是**推理纪律**，需要强制推理步骤
- **诊断错误比问题更危险**：如果用错解法，会浪费资源且无法解决问题

## 使用方式

### 被Skill调用
```markdown
## Step: 确定法律关系

请使用法条库分析本案法律关系：

1. 读取 [[prompt-法条调用机制]] 了解调用规则
2. 读取 [[婚姻财产纠纷-核心法条]] 的索引表
3. 根据案件事实，精读相关法条章节
4. 必须引用到"条"级别
5. 将法条与案件事实一一对应
```

### 被Agent调用
Agent可以根据案件类型自动选择法条库：
- 婚姻财产 → `婚姻财产纠纷-核心法条.md`
- 合同纠纷 → `合同纠纷-核心法条.md`（待建）
- 劳动争议 → `劳动争议-核心法条.md`（待建）

## 扩展路径

添加新案件类型的法条库非常简单：

1. **创建法条库文件**：`{案件类型}-核心法条.md`
2. **在索引表添加一行**：记录法条覆盖范围
3. **测试验证**：通过A/B测试验证效果

## 与其他组件的关系

```
┌──────────────┐
│   用户案例   │
└──────┬───────┘
       │
       ▼
┌──────────────┐     ┌──────────────┐
│   Skill/     │────▶│   法条库     │
│   Agent      │     │  (弹药)      │
│  (枪)        │◀────┤              │
└──────────────┘     └──────────────┘
       │
       ▼
┌──────────────┐
│  法律分析    │
│  (有依据)    │
└──────────────┘
```

- **法条库是"弹药"**：提供法律条文依据
- **Skill是"枪"**：提供强制引用规则和推理框架
- **两者配合**：才能产生准确的法律分析

## 维护原则

1. **法条准确性**：确保引用最新有效版本
2. **结构一致性**：所有法条库文件使用相同格式
3. **渐进式调用**：始终通过索引表按需读取
4. **测试验证**：每个新法条库都必须通过A/B测试

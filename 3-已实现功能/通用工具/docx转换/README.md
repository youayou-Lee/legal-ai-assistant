# 法律文书docx转换工具

> 将AI生成的纯文本法律文书转换为格式规范的Word文档。

## 部署位置

`~/.agents/scripts/text-to-docx.py`

## 用法

```bash
python3 ~/.agents/scripts/text-to-docx.py input.txt output.docx
```

被三个文书Skill（起诉状/答辩状/上诉状）在最后一步调用。

## 格式规范

基于[法院诉讼文书格式标准](http://jningssfy.sdcourt.gov.cn/)和[律师文书排版指引](https://zhuanlan.zhihu.com/p/358836570)：

| 项目 | 设置 |
|------|------|
| 纸张 | A4 (21x29.7cm) |
| 页边距 | 上3.3cm / 下3.2cm / 左2.7cm / 右2.4cm |
| 正文字体 | 仿宋_GB2312 三号(16pt) |
| 标题字体 | 黑体 三号 加粗 居中 |
| 行距 | 固定值28磅 |
| 首行缩进 | 0.74cm（正文段落） |

## 段落类型自动识别

脚本自动识别10种段落类型并设置对应格式：

| 类型 | 缩进 | 对齐 | 加粗 | 示例 |
|------|------|------|------|------|
| 标题 | 无 | 居中 | 是 | 民事上诉状 |
| 段落标题 | 无 | 左 | 是 | 上诉请求、事实与理由 |
| 当事人信息 | 无 | 左 | 否 | 上诉人（原审被告）：杨刚... |
| 一级标题 | 有 | 左 | 是 | 一、上诉人与被上诉人... |
| 正文 | 有 | 左 | 否 | 具体论述内容 |
| 此致 | 无 | 左 | 否 | 此致 |
| 法院名 | 无 | 左 | 否 | 南宁市中级人民法院 |
| 签名 | 无 | 右 | 否 | 上诉人：（签名） |
| 日期 | 无 | 右 | 否 | 2026年4月7日 |
| 附项 | 无 | 左 | 否 | 附： |

## 清理功能

自动清理AI输出中的Markdown残留：
- 横线（`---`、`===`）
- 标题符号（`#`）
- 加粗/斜体（`**`、`*`）
- 表格标记（`|`）
- 代码块（` ``` `）
- Checkbox（`- [ ]`）

## 关键技术点

- **东亚字体渲染**：通过XML设置`w:rFonts`的`w:eastAsia`属性，确保中文字体在Word中正确显示
- **签名区检测**：法院名之后自动进入签名区，后续行识别为签名/日期并右对齐
- **日期行精确匹配**：只匹配纯日期行（`^...日$`），避免把"2024年1月1日，被告..."误识别为日期

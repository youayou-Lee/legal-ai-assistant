# 法言法语优化共享约束层

## 技术方案
共享Markdown约束文件，被所有法律文书Skill引用

## 一句话说明
统一的法律文书法言法语标准，覆盖术语替换、句式模板、金额日期格式、法条引用、段落结构5个维度。

## 部署位置
- 约束层源文件：`~/.agents/skills/shared/法言法语约束.md`
- 范例资料：`0-参考资料/法律文书范例/`（3份）

## 集成方式

OpenClaw Skill不支持运行时文件include，所以采用**内联嵌入**方式：
1. `shared/法言法语约束.md` 是唯一真实来源（source of truth）
2. 各Skill的`SKILL.md`中"法言法语"段内联嵌入关键约束（术语替换要点+句式模板+格式规范）
3. 各Skill中标注共享文件路径，供人工查阅完整版

修改共享约束层后，需手动同步到各Skill的内联内容。具体修改了哪些文件的哪些段落见 [[结果]]。

## 背景

吴律要求AI输出使用法律专业语言（"法言法语"）。诊断发现现有3个文书Skill的法言法语约束仅覆盖术语替换（各4对）和格式禁止项，缺失句式引导、逻辑连接词、金额日期格式、法条引用规范等关键维度。

通过A/B对比测试（同一民间借贷案例，优化前约束 vs 法言法语标准），识别出5个核心缺口：

| 维度 | 优化前评分 | 根因 |
|------|-----------|------|
| 术语替换 | 4/10 | 仅4对替换表，覆盖面<10% |
| 句式模板 | 3/10 | 无正面引导，只有禁止项 |
| 金额/日期格式 | 3/10 | 完全空白 |
| 法条引用 | 1/10 | 无"必须具体到条款号"要求 |
| 段落结构 | 5/10 | 缺标准段落指引 |

## 核心内容

共享约束层包含6个部分：

1. **术语替换表（30+对）** — 分4类：诉讼行为类、法律关系类、主观评价类、诉讼请求类
2. **关键句式模板（8个）** — 合同建立、违约事实、法条引用三段论、清偿抵充、起诉状/答辩状/上诉状结尾段、时间段表述
3. **金额与日期格式规范** — "人民币100,000元"千分位格式、"XXXX年XX月XX日"标准日期
4. **法条引用规范** — 必须具体到法律名称+条款号，禁止"相关规定"；三段论论证（法律规定→事实对应→法律结论）
5. **"事实与理由"段落结构指引** — 分别为起诉状、答辩状、上诉状提供标准段落模板
6. **通用质量检查清单** — 10项自检

## 已集成的Skill

| Skill | 更新内容 |
|-------|---------|
| civil-complaint（起诉状） | 替换原4对术语表+2行约束为完整5维度约束 |
| civil-defense-statement（答辩状） | 同上，增加答辩状专用开场白和结尾句式 |
| civil-appeal（上诉状） | 同上，增加上诉状专用论证句式 |

## 优化迭代

- **v1**（2026-04-08初版）：基于诊断测试+真实范例创建，5维度覆盖
- **v1.1**（2026-04-08迭代）：修复验证中发现的残留问题
  - 新增：禁止"此后未再支付任何款项"→"此后再未履行任何偿付义务"
  - 新增：禁止句首转折词"然/但是"→直接用"尚欠"陈述精确金额
  - 优化：违约事实句式模板增加"尚欠"精确金额表述指引

## ⚠️ 验证状态

- **测试环境**：Claude Code中模拟测试（民间借贷案例A/B对比）
- **测试结果**：5维度平均分从3.2/10提升至8.9/10，残留2-3处轻度口语化已修复
- **OpenClaw端到端运行**：❌ 未在OpenClaw中实际运行验证
- **律师验证**：❌ 吴律未确认优化后的输出是否达标
- **验证场景**：仅民间借贷纠纷一种，其他民事类型未测试

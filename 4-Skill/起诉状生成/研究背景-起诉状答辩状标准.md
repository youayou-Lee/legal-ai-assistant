# 研究背景：起诉状与答辩状的法律标准

> 本文档基于《中华人民共和国民事诉讼法》及相关司法解释，整理起诉状和答辩状的法定格式、质量标准和实务要求。

---

## 一、起诉状法定格式

### 法律依据

| 法条来源 | 条文编号 | 核心内容 |
|----------|----------|----------|
| 《民事诉讼法》 | 第122条 | 起诉应当向人民法院递交起诉状，并按照被告人数提出副本 |
| 《民事诉讼法》 | 第121条（2023修正前为第121条） | 起诉状应当记明的事项 |
| 最高人民法院 | 《民事起诉状、答辩状示范文本》（2024年3月4日起试行） | 最新示范文本规范 |

**重要更新（2024年）**：最高人民法院商司法部、中华全国律师协会于2024年3月4日起试行《民事起诉状、答辩状示范文本》，试行期一年。

### 必须记明的事项（第121条，2023年修正版）

1. 原告的姓名、性别、年龄、民族、职业、工作单位、住所、联系方式
2. 被告的姓名、性别、工作单位、住所等信息
3. 诉讼请求和所根据的事实与理由
4. 证据和证据来源，证人姓名和住所

### 标准格式结构

```
民事起诉状

原告：XXX，男/女，XXXX年XX月XX日出生，XX族，住XXXX，联系电话XXXX

被告：XXX，男/女，XXXX年XX月XX日出生，XX族，住XXXX，联系电话XXXX

诉讼请求：
1. XXXXXXXX
2. XXXXXXXX
3. 本案诉讼费用由被告承担

事实与理由：
XXXXXX（按时间顺序叙述事实）
XXXXXX（阐述法律依据）

此致
XX市XX区人民法院

具状人：XXX
XXXX年XX月XX日
```

### 质量标准

1. **诉讼请求明确具体**：金额确定、履行方式明确
2. **事实陈述清晰**：时间、地点、人物、事件完整
3. **法律关系准确**：正确认定民事法律关系性质
4. **管辖法院正确**：符合地域管辖和级别管辖规定
5. **证据链完整**：每项事实有证据支撑
6. **法言法语规范**：使用规范法律术语

## 二、答辩状法定格式

### 法律依据

《民事诉讼法》第125条：人民法院应当在立案之日起五日内将起诉状副本发送被告，被告应当在收到之日起十五日内提出答辩状。

### 答辩状的法定要求

被告不提出答辩状的，**不影响人民法院审理**。但提出答辩状有以下作用：

1. 阐明被告观点，帮助法官查明事实
2. 明确争议焦点，提高庭审效率
3. 保护被告合法权益

### 标准格式结构

```
民事答辩状

答辩人：XXX，男/女，XXXX年XX月XX日出生，XX族，住XXXX，联系电话XXXX

因原告XXX诉被告XXX [案由] 一案，提出答辩如下：

对原告陈述的XXXX事实，无异议。

对原告陈述的XXXX事实，有异议。实际情况是：XXXXXX。

对原告的第X项诉讼请求，答辩人认为不应支持，理由如下：
1. XXXXXX
2. XXXXXX

综上所述，请求法院驳回原告的全部/部分诉讼请求。

此致
XX市XX区人民法院

答辩人：XXX
XXXX年XX月XX日
```

### 答辩策略分类

#### 1. 程序性抗辩

- **管辖异议**：法院无管辖权
- **诉讼时效**：原告起诉已超过诉讼时效
- **主体资格**：原告或被告主体不适格
- **一事不再理**：本案已经法院裁判

#### 2. 实体性抗辩

- **事实否认**：原告陈述的事实与实际不符
- **法律关系抗辩**：双方不存在原告诉称的法律关系
- **履行抗辩**：被告已履行义务/原告违约在先
- **金额异议**：原告诉请金额计算错误
- **责任免除**：存在不可抗力、对方过错等免责事由

### 质量标准

1. **逐项答辩**：对每项诉讼请求必须有明确答辩意见
2. **事实清楚**：陈述被告方掌握的事实版本
3. **理由充分**：抗辩理由有法律依据支撑
4. **证据支撑**：关键抗辩有证据证明
5. **逻辑严密**：答辩观点前后一致
6. **法言法语规范**

## 三、律师实务审核标准（来自吴律需求文档）

### 律师希望只需审核的核心内容

1. 当事人是否适格或是否遗漏当事人
2. 是否遗漏诉讼请求、答辩要点是否全面
3. 事实理由是否完整、准确
4. 法律适用是否准确
5. 是否已形成完整证据链、证明目的是否清晰

### AI产出的必要性评价（吴律）

**有必要（核心价值）**：
- 文书及证据目录生成
- 风险分析、诉讼策略分析
- 法条匹配
- 庭审提纲（含争议焦点、发问清单）

**无必要（干扰信息）**：
- 过多展示过程性的东西
- 简单问题复杂化，堆砌无必要的文字

## 四、关键设计约束

### 约束1：推理纪律

AI缺少法律推理步骤意识，必须通过**强制推理步骤模板**约束：

1. 先拆解原告主张
2. 再分析事实争议
3. 然后识别抗辩点
4. 最后组织答辩

### 约束2：法条引用

分析结论**必须包含**：
1. 法律性质认定
2. 法条依据（至少到"条"的级别）
3. 计算公式和过程（如涉及金额）

### 约束3：法言法语

- 使用规范法律术语
- 避免口语化表达
- 符合文书格式要求

## 七、常见错误清单（补充）

### 7.1 起诉状常见错误

| 错误类型 | 具体表现 | 后果 |
|----------|----------|------|
| **形式错误** | 文书抬头写成"上诉状""告状书"等 | 立案驳回 |
| **信息遗漏** | 当事人信息不完备，缺少联系电话 | 送达困难 |
| **请求不明** | 诉讼请求模糊，无法确定具体内容 | 法院要求补正 |
| **遗漏请求** | 未主张全部权利 | 权益受损 |
| **事实矛盾** | 事实与理由与诉讼请求不一致 | 逻辑混乱 |
| **过度陈述** | 陈述无关事实，暴露漏洞 | 被对方利用 |
| **法律适用错误** | 适用法律与案件性质明显不符 | 主张不被支持 |
| **自认不利事实** | 在起诉状中承认对己不利事实 | 无需对方举证即成立（民诉证据规定第3条） |

### 7.2 答辩状常见错误

| 错误类型 | 具体表现 | 后果 |
|----------|----------|------|
| **逾期提交** | 超过15天答辩期限 | 影响答辩权行使 |
| **缺乏针对性** | 未针对原告诉讼请求逐一回应 | 答辩无效 |
| **表态不明** | 未明确承认、否认或部分承认 | 争议焦点不清 |
| **事实自认** | 在答辩状中承认原告全部主张 | 丧失抗辩机会 |
| **法律依据缺失** | 仅陈述事实，无法律依据支撑 | 说服力不足 |
| **反诉超期** | 反诉未在答辩期限内一并提出 | 反诉不被受理 |

## 八、法言法语用词规范（补充）

### 8.1 起诉状常用表述

| 场景 | 规范表述 | 避免使用 |
|------|----------|----------|
| 开篇 | "原告与被告[纠纷类型]一案，向贵院提起诉讼" | "我要告他..." |
| 事实陈述 | "原告与被告于[时间][行为]" | "他当时..." |
| 侵权描述 | "被告的行为已构成[侵权类型]，严重侵害了原告的合法权益" | "他欺负人..." |
| 请求表述 | "请求判令被告[具体履行内容]" | "让他赔钱..." |
| 结尾 | "恳请贵院查明事实，依法支持原告的全部诉讼请求" | "请法官主持公道..." |

### 8.2 答辩状常用表述

| 场景 | 规范表述 |
|------|----------|
| 开篇 | "因[案号]一案，现提出答辩如下" |
| 承认 | "对原告主张的[某事实]，被告无异议" |
| 否认 | "原告主张的[某事实]与客观事实不符" |
| 抗辩 | "原告的诉讼请求缺乏事实和法律依据" |
| 反诉引入 | "被告认为，原告的行为已构成[侵权]，依法应承担相应责任" |

## 十一、补充研究成果（基于最新Web搜索）

### 11.1 起诉状技术格式规范（来自搜索结果）

**字体和排版标准：**
- 标题：黑体2号，居中
- 正文：仿宋小三号
- 行间距：1.5倍行距
- 建议参照法院判决书的格式标准

**版面要求：**
- 文书各部分要素表述完整、规范
- 事实与理由、诉讼请求等部分标题需加粗标识
- 按被告人数提交副本（每个被告一份）

### 11.2 答辩状策略深化（基于实务研究）

**答辩三层次理论（无讼平台）：**

1. **诉讼要件的答辩**（程序性答辩）
   - 管辖权异议
   - 主体适格性异议
   - 重复起诉异议
   - 仲裁前置程序未完成
   - 诉讼时效届满

2. **请求权要件事实的否定答辩**
   - 针对原告请求权基础的否定
   - 否认构成要件事实的存在

3. **抗辩规范要件事实的答辩**
   - 主张抗辩权成立的事实
   - 权利障碍抗辩、权利消灭抗辩、抗辩权

**逐条答辩的具体操作（法官建议）：**

针对起诉状的"事实与理由"部分，应明确选择：
- **承认** - 对无异议的事实予以承认
- **否认** - 对不属实的事实予以否认（**必须附理由及证据**）
- **抗辩** - 提出实质性抗辩内容

### 11.3 律师实务经验补充

**起诉前准备清单：**
1. 充分调查被告身份信息和财产状况
2. 诉讼请求设计要周全，考虑所有可能的请求
3. 事实陈述简洁清晰，按时间顺序陈述
4. 注意诉讼时效，确保在时效内起诉
5. 证据准备充分，形成完整证据链

**答辩策略要点：**
1. 针锋相对，焦点突出
2. 否认的事项必须附理由及证据，不能空口否认
3. 抗辩需具备实质性内容，不能流于形式
4. 层次清晰：先表明我方主张，再进行论证
5. 注意15天答辩期限，预留送达时间

### 11.4 答辩失权风险提示

根据实务经验：
- 未在答辩中否认可能视为自认（参考美国联邦规则）
- 不提交答辩状不影响法院审理，但可能丧失异议机会
- 部分律师选择不提交书面答辩以保留庭审"突袭"效果（需谨慎）

## 十二、AI生成文书的特别注意事项

### 12.1 推理纪律强制约束

AI缺乏法律推理步骤意识，必须通过**强制推理步骤模板**约束：

**起诉状推理流程：**
```
1. 识别案件性质 → 确定法律关系
2. 提取关键事实 → 按时间线组织
3. 匹配法律条文 → 确定请求权基础
4. 设计诉讼请求 → 确保明确具体
5. 组织事实与理由 → 事实与法律对应
6. 整理证据清单 → 确保证明目的清晰
```

**答辩状推理流程：**
```
1. 解析起诉状 → 提取诉讼请求和事实主张
2. 识别程序性答辩点 → 管辖、时效、主体资格
3. 分析实体性答辩点 → 承认/否认/抗辩选择
4. 匹配法律依据 → 支持答辩观点
5. 组织逐条回应 → 针对每项请求回应
6. 生成证据建议 → 支持否认和抗辩
```

### 12.2 法条引用准确性要求

分析结论**必须包含**：
1. 法律性质认定（如：民间借贷纠纷、买卖合同纠纷）
2. 法条依据（至少到"条"的级别，理想到"款"）
3. 计算公式和过程（如涉及金额）
4. 法条有效性验证（避免引用失效法条）

### 12.3 多模型交叉验证机制

针对关键内容进行交叉验证：
- **法律适用准确性**：使用多个模型验证法条匹配
- **事实认定准确性**：交叉验证事实提取是否完整
- **诉讼请求完整性**：验证是否遗漏重要请求
- **答辩策略合理性**：验证承认/否认/抗辩选择是否恰当

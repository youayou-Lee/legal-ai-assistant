# 审查Agent — 后续方向

## 短期（1-2周）

### 1. 用真实AI模型跑完整闭环
- 用OpenClaw的reviewer-deepseek和reviewer-qwen跑v2测试案例
- 对比模拟vs真实的差异
- **与P0问题1对应**

### 2. 部署到律师端
- 打包配置文件（3个AGENTS.md）
- 部署到吴律/黄律的OpenClaw
- 用真实案例跑一次完整闭环
- **与P0问题2对应**

### 3. 收集律师对修正后版本的反馈
- 重点验证：金额重算是否正确、法条补充是否准确
- 这决定闭环修正协议是否靠谱

## 中期（1-2月）

### 4. 答辩状检查清单验证
- 起诉状清单已测试，答辩状清单已设计未测试
- 用真实答辩状测试DeepSeek的5项答辩状检查清单
- 根据结果调整

### 5. 扩展到更多案件类型
- 当前仅民间借贷。需扩展：买卖合同、劳动争议、婚姻财产
- 每种类型需要专门的检查清单子项
- 参考 [[../详细诊断-合同纠纷]]、[[../详细诊断-婚姻财产]] 中的弱点分析

### 6. 检查清单遵循率优化
- 观察真实AI是否逐项执行
- 如果遵循率低：精简清单、突出关键项
- 如果遵循率高：可考虑增加维度

## 长期（3-6月）

### 7. 审查结果持久化
- 将审查历史保存到案件档案
- 支持多次审查对比
- 与 **案件管理** 功能联动

### 8. 增量审查
- 文书修改后只审查变更部分
- 降低审查成本和延迟

### 9. 第三个视角（己方律师自查）
- 当前两方（对方律师+法官）
- 可增加第三个模型从己方律师视角自查
- 评估三方验证是否显著优于两方

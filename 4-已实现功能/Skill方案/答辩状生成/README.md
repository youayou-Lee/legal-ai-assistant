# 民事答辩状生成 - 功能总览

## 功能简介

**功能名称**：民事答辩状生成

**技术方案**：OpenClaw Skill（SKILL.md + 8步强制推理框架，含借款合同专项检查Step 4.5）

**一句话说明**：根据原告起诉状和被告方材料，按强制推理步骤生成符合法定格式的民事一审被告答辩状

---

## 核心特性

### 1. 对称设计：起诉状 vs 答辩状

| 维度 | 起诉状生成 | 答辩状生成 |
|------|-----------|-----------|
| **立场** | 原告角度 | 被告角度 |
| **核心目标** | 证据链完整 + 金额准确 | 逐条反驳 + 抗辩策略选择 |
| **独特功能** | 诉讼请求设计、证据链组织 | Step 4.5 借款合同专项检查 |
| **推理重点** | 构建完整事实叙事 | 识别对方漏洞和抗辩点 |

### 2. 8步强制推理框架

```
Step 1: 拆解原告诉讼请求
   ↓
Step 2: 提取原告事实主张
   ↓
Step 3: 检查程序性答辩点（管辖、时效、主体资格）
   ↓
Step 4: 分析实体性答辩点（承认/否认/抗辩）
   ↓
Step 4.5: 【借款合同专项检查】砍头息、期限不明、利息约定不明
   ↓
Step 5: 组织逐条回应
   ↓
Step 6: 生成答辩理由（法条引用）
   ↓
Step 7: 整理证据建议
   ↓
Step 8: 输出法定格式答辩状
```

### 3. 借款合同专项检查（Step 4.5）

答辩状独有的专项检查模块，主动发现原告起诉状中的问题：

- **砍头息识别**：借款本金 vs 实际到账金额
- **借款期限分析**：约定期限 vs 期限不明的法律后果
- **利息约定检查**：明确约定 vs 约定不明的法律后果
- **已还款抵充**：先息后本 vs 先本后息

## 与起诉状生成的对比

### 功能互补

| 功能 | 起诉状 | 答辩状 |
|------|-------|-------|
| **输出主体** | 原告 | 被告 |
| **输入材料** | 原告方材料 + 案情描述 | 起诉状 + 被告方材料 |
| **推理起点** | 从零构建事实叙事 | 从对方叙事中找漏洞 |
| **策略选择** | 如何最有利地陈述事实 | 承认/否认/抗辩 |
| **法条重点** | 请求权基础 | 抗辩权基础 |

### Step 4.5 的独特性

**为什么答辩状需要Step 4.5，而起诉状不需要？**

- 起诉状：站在主动方，只需确保自己的主张完整、准确
- 答辩状：站在被动方，需要主动**发现对方起诉状中的问题**

例如民间借贷纠纷：
- 原告主张借款50万 → 被告需要主动识别砍头息（实际到账45万）
- 原告主张借款期限1年 → 被告需要主动分析期限不明的法律后果
- 原告主张年利率12% → 被告需要检查利息约定是否明确

## 技术架构

### OpenClaw Skill配置

```yaml
技能名称: civil-defense-statement
触发词: "生成答辩状" / "民事答辩状"
输入要求:
  - 原告起诉状（必需）
  - 被告方陈述材料（必需）
  - 被告方证据清单（可选）
输出格式:
  - 民事答辩状（法定格式）
  - 答辩策略分析（附注）
  - 证据建议清单（附注）
推理约束:
  - 8步强制推理框架
  - 逐条回应要求
  - 法条引用强制
```

### 质量保证机制

1. **逐条答辩强制检查**：每项诉讼请求必须明确回应
2. **法条引用验证**：抗辩理由必须有明确法条支撑
3. **逻辑一致性检查**：承认/否认/抗辩选择必须前后一致
4. **借款合同专项检查**：Step 4.5自动触发

## 快速开始

### 使用方式

1. 准备材料：
   - 原告起诉状
   - 被告方陈述材料（事实版本）
   - 被告方证据清单（如有）

2. 触发Skill：
   ```
   /civil-defense-statement
   或
   "请生成答辩状，基于以下起诉状和被告材料..."
   ```

3. 获得输出：
   - 民事答辩状（可直接提交法院）
   - 答辩策略分析（供律师参考）
   - 证据建议清单（待补充证据提示）

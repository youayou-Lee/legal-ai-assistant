# 对方策略预判模块

## 功能概述

**功能名称**：对方策略预判

**技术方案**：OpenClaw Skill

**一句话说明**：站在对方律师视角，系统推演对方可能采取的答辩策略、质证意见和辩论重点

**核心理念**：强制换位思考，通过6步推理框架约束AI站在对方立场分析

**当前状态**：v1已完成测试，待真实案例验证

**部署位置**：`~/.agents/skills/opponent-prediction/`

## 核心价值

律师在庭审准备中最容易犯的错误是"只看自己的案子，不看对方的牌"。预判模块通过强制换位思考的6步推理框架，帮助律师：

- 📋 **系统化预判**：覆盖答辩点、质证意见、辩论重点三个维度
- 🎯 **发现盲区**：识别自身案件的薄弱环节
- 💡 **提前准备**：针对对方可能的主张提前准备应对方案
- 📊 **概率评估**：按高/中/低概率排序，优先防范高风险点

## 技术架构

### 实现方式

使用 OpenClaw 原生 Skill 配置，零代码实现。

- **Skill 文件**：`~/.agents/skills/opponent-prediction/SKILL.md`
- **Agent 配置**：无需独立 Agent，通过主 Agent 调用
- **模型要求**：支持任何 OpenClaw 兼容的 LLM（推荐 Claude 3.5 Sonnet 或 GPT-4）

### 6步强制推理框架

| Step | 内容 | 关键约束 |
|------|------|---------|
| 0 | 锁定用户/对方身份 | 必须先确认原告还是被告 |
| 1 | 从对方视角重构事实 | 全部站在对方立场 |
| 2 | 预判答辩点（5维度） | 诉讼主体+管辖+时效+事实+法律 |
| 3 | 预判质证意见 | 按证据三性攻击 |
| 4 | 预判辩论重点 | 争议焦点+论证逻辑 |
| 5 | 盲区扫描+应对建议 | 强制找自己方案的弱点 |
| 6 | 自检 | 对照检查清单 |

## 测试验证

### v1 测试结果（模拟案例）

使用民间借贷纠纷案例测试，全部验证点命中：

- ✅ **答辩点预判** 5/5：砍头息抗辩、已还款抗辩、诉讼时效、利息基数、LPR上限
- ✅ **质证意见** 3/3：借款协议、转账凭证、微信聊天记录
- ✅ **辩论重点** 3/3：本金认定、还款性质、利息起算点
- ✅ **盲区发现** 5个（2高风险 + 3中等风险）

## 支持的案件类型

当前支持以下4种民事案件类型：

1. **民间借贷纠纷**
2. **合同纠纷**
3. **婚姻财产纠纷**
4. **劳动争议**

每种案件类型都有针对性的预判重点，详见 SKILL.md 的"特殊案件类型的预判重点"章节。

## 使用方法

### 触发方式

在 OpenClaw Gateway 中直接描述需求：

```
帮我预判一下对方会怎么答辩
分析对方的答辩点和质证意见
站在被告视角分析我的案件风险
```

### 输入要求

需要提供以下任意一种材料：

- 起诉状/答辩状
- 案件事实陈述
- 证据清单

系统会自动识别用户身份（原告方/被告方），如无法判断会提示确认。

### 输出内容

输出包含以下7个部分：

1. **身份确认**：用户方和对方方的身份锁定
2. **对方视角事实重构**：对方认可/否认/可能主张的事实
3. **对方答辩点预判**：程序性抗辩 + 实体性抗辩（含概率评估）
4. **对方质证意见预判**：对用户方每份证据的攻击策略
5. **对方辩论重点预判**：按优先级排列的核心论点
6. **盲区扫描与应对建议**：高风险/中等风险盲区 + 具体应对方案
7. **质量自检**：对照检查清单确认完整性

## 设计思考

详见 [[思考]]

**核心洞察**：AI天然倾向站在用户视角分析，必须通过 Step 0 身份锁定和 Step 6 视角自检来约束。

**设计取舍**：为什么按"概率+法条+推理过程"三要素描述预判点而非简单列表？

**方法论验证**：验证了 Harness Engineering 的核心理念——"输出不好时，改 Harness 不改输出"。

## 当前局限

详见 [[存在的问题]]

- ⚠️ 仅使用模拟案例测试，尚未用真实案例验证
- ⚠️ 概率评估是定性判断（高/中/低），未经过数据校准
- ⚠️ 仅支持民事案件，刑事案件适配逻辑未开发
- ⚠️ 案件类型覆盖仅4种，实际案件类型远多于此
- ⚠️ 与起诉状/答辩状生成功能尚未联动

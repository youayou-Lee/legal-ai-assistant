# 法条库测试案例与方法

## 测试概述

法条库的有效性通过两类测试验证：
1. **A/B测试**：对比有无法条库的分析质量差异
2. **边界探测测试**：区分"知识缺口"和"推理纪律缺口"

---

## A/B测试设计

### 测试目的
验证法条库对婚姻财产纠纷案件分析质量的提升效果。

### 测试领域
**婚姻财产纠纷** - 选择理由：
- 法条数量多且复杂（民法典+司法解释）
- 涉及金额大，律师收费高
- AI法律助手常见应用场景
- 法条更新频繁，需要准确引用

### 测试案例设计

#### 案例1：婚前房产婚后增值
**案情**：
- 婚前男方首付购房，登记在自己名下
- 婚后共同还贷20万元
- 房产现值300万元，增值150万元
- 争议：婚后增值部分如何分割？

**关键法条**：
- 民法典第1063条（个人财产范围）
- 最高人民法院司法解释第78条（婚后增值部分归属）

**测试点**：
- AI是否正确识别"婚后共同还贷部分对应增值"为共同财产
- 是否能计算出女方应得份额
- 是否引用具体法条编号

#### 案例2：父母出资购房
**案情**：
- 婚后男方父母出资200万元购房
- 登记在男方名下
- 争议：这是夫妻共同财产还是男方个人财产？

**关键法条**：
- 民法典第1062条（夫妻共同财产范围）
- 最高人民法院司法解释第29条（父母出资认定）

**测试点**：
- AI是否区分"赠与双方"和"赠与一方"
- 是否考虑"没有明确表示"的默认规则
- 是否引用最新司法解释（2021年修正）

#### 案例3：夫妻共同债务认定
**案情**：
- 婚前男方借款50万元赌博
- 婚后女方不知情
- 债权人要求女方共同偿还

**关键法条**：
- 民法典第1064条（夫妻共同债务三种情形）
- 最高人民法院司法解释第33条（婚前债务归属）

**测试点**：
- AI是否正确适用民法典第1064条第2款（例外情形）
- 是否区分"用于夫妻共同生活"和"未用于"
- 是否引用到"条"级别

#### 案例4：婚后购房一方父母出资
**案情**：
- 婚后购买房产，总价500万
- 女方父母出资300万，男方出资200万
- 登记在双方名下
- 争议：离婚时如何分割？

**关键法条**：
- 民法典第1087条（离婚财产分割原则）
- 最高人民法院司法解释第76条（出资比例与产权比例）

**测试点**：
- AI是否考虑出资比例
- 是否适用"照顾子女和女方权益"原则
- 是否给出具体分割方案

#### 案例5：婚前协议的效力
**案情**：
- 婚前签订财产协议
- 约定"婚后各自收入归各自所有"
- 婚后女方全职在家照顾家庭
- 争议：协议是否有效？

**关键法条**：
- 民法典第1065条（夫妻财产约定）
- 民法典第1088条（家务劳动补偿）

**测试点**：
- AI是否识别夫妻财产约定的效力
- 是否考虑"显失公平"情形
- 是否提及家务劳动补偿请求权

### 对照组（A组）设置

**条件**：
- 不注入法条库
- 不提示必须引用法条
- 仅凭AI自身训练数据中的法律知识回答

**测试指令**：
```
请分析以下婚姻财产纠纷案件的法律问题，给出你的专业意见。

[案情描述]

请回答：
1. 本案涉及哪些法律关系？
2. 相关的法律规定是什么？
3. 法院可能如何判决？
```

### 实验组（B组）设置

**条件**：
- 注入婚姻财产纠纷法条库全文
- 强制要求引用法条到"条"级别
- 必须将法条与案件事实一一对应

**测试指令**：
```
请使用法条库分析以下婚姻财产纠纷案件：

第一步：阅读 [[婚姻财产纠纷-核心法条]] 的索引表
第二步：根据案件事实，精读相关法条章节
第三步：分析本案并回答：
1. 引用相关法条（必须到"条"级别）
2. 将法条与案件事实一一对应
3. 给出具体的法律结论

重要规则：
- 不得凭常识推断，必须有法条依据
- 法条引用必须准确到"条"
- 不确定的法条必须标注"不确定"
```

### 评估标准

#### 1. 法条引用覆盖率
- **定义**：应该引用的法条中，实际引用的比例
- **评分**：
  - 100%：完整覆盖所有相关法条
  - 50-99%：部分覆盖
  - <50%：严重缺失
  - 0%：未引用任何法条

#### 2. 法条引用准确性
- **定义**：引用的法条编号是否正确，内容是否准确
- **评分**：
  - 准确：法条编号和内容完全正确
  - 部分准确：法条编号正确，但理解有偏差
  - 错误：引用了无关法条或编造法条

#### 3. 结论准确性
- **定义**：法律结论是否符合司法实践
- **评分**：
  - 准确：与法院判例一致
  - 基本准确：大方向对，细节有瑕疵
  - 不准确：主要结论错误

#### 4. "不确定"回答数
- **定义**：AI回答"不确定"、"无法判断"的次数
- **评分**：次数越少越好

#### 5. 分析深度
- **定义**：是否深入分析每个法条的适用条件
- **评分**：
  - 深度：逐条分析，考虑适用条件
  - 中等：提及法条但未深入
  - 表面：仅给结论，无分析过程

## 边界探测测试

### 测试目的
区分AI在不同案件类型中的"知识缺口"和"推理纪律缺口"，确定法条库的适用范围。

### 测试领域
选择三类常见案件：
1. **合同纠纷**（高频，但法条相对简单）
2. **婚姻财产纠纷**（法条复杂，司法解释多）
3. **劳动争议**（专业性较强，法规分散）

### 测试方法

#### 第一轮：知识缺口测试
**测试指令**：
```
请回答以下法律专业问题（仅凭你的训练数据）：

1. 民法典第1064条规定的"夫妻共同债务"包括哪三种情形？
2. 根据民法典第563条，当事人可以解除合同的五种法定情形是什么？
3. 劳动法第39条规定，劳动者有下列情形之一的，用人单位可以解除劳动合同？

请尽可能准确地回答。如果不确定，请标注"不确定"。
```

**评估**：
- 完全准确回答 → 无知识缺口
- 部分准确 → 知识缺口
- 回答"不确定"或编造 → 严重知识缺口

#### 第二轮：推理纪律测试
**测试指令**：
```
请分析以下案例，并逐步推理：

[提供完整案情和正确法条]

请按以下格式回答：
Step 1: 识别案件事实
Step 2: 匹配相关法条
Step 3: 分析法条适用条件
Step 4: 逐一对照事实与条件
Step 5: 得出法律结论

注意：即使你知道结论，也要展示推理过程。
```

**评估**：
- 是否跳过推理步骤直接给结论 → 推理纪律缺口
- 是否逐项对照事实与法条 → 推理纪律良好
- 是否遗漏关键事实 → 推理纪律缺口

### 测试结果

#### 合同纠纷（10个案例）

| 指标 | 结果 |
|------|------|
| 知识缺口占比 | 30% |
| 推理纪律缺口占比 | 70% |
| 典型错误 | 跳过"要约-承诺"分析，直接判断合同成立 |
| 解法 | 强制推理步骤（Skill Prompt），法条库帮助有限 |

**典型案例**：
- 案情：买方发邮件订购100台电脑，卖方回复"收到"，后卖方反悔
- AI错误：直接判定"合同已成立"，未分析邮件是否构成有效承诺
- 正确分析：应先判断"收到"是否构成承诺（需明确表示接受）

#### 婚姻财产纠纷（10个案例）

| 指标 | 结果 |
|------|------|
| 知识缺口占比 | 70% |
| 推理纪律缺口占比 | 30% |
| 典型错误 | 编造法条编号、混淆婚前婚后财产规则 |
| 解法 | **法条库是关键解法** |

**典型案例**：
- 案情：婚后男方父母出资购房，登记在男方名下
- AI错误：凭常识判断"父母出资是给儿子的"，未引用任何法条
- 正确分析：应引用司法解释第29条，区分"赠与双方"和"赠与一方"

## 测试局限性

### 已知局限

1. **样本量较小**：每个类型仅10个案例，统计显著性不足
2. **案例选择偏差**：选用的都是典型争议案例，可能不代表整体分布
3. **评估主观性**：部分评估标准（如"分析深度"）依赖人工判断
4. **单一模型测试**：仅在Claude Sonnet上测试，未验证其他模型

### 改进方向

1. **扩大样本量**：每个案件类型测试50+案例
2. **引入第三方评估**：邀请执业律师盲评
3. **多模型验证**：在GPT-4、文心一言等模型上复现测试
4. **建立量化指标**：减少主观评估，使用可量化指标

---

## 测试结论

### 核心发现
1. **婚姻财产纠纷**：法条库效果显著，知识缺口占比70%
2. **合同纠纷**：推理纪律是关键，法条库帮助有限
3. **诊断错误比问题更危险**：用错解法会浪费资源

### 推广建议
- **优先领域**：婚姻财产、继承纠纷等法条密集型领域
- **谨慎领域**：合同纠纷等推理密集型领域（需配合强制推理步骤）
- **待验证**：劳动争议、知识产权、行政法等领域

### 后续测试
- 扩大到合同纠纷法条库测试
- 验证法条库在起诉状/答辩状生成中的效果
- 测试多模型交叉验证机制

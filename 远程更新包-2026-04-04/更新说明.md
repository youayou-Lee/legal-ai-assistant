# OpenClaw 架构更新 - 远程部署指南

> 本文档供 OpenClaw 阅读并执行更新操作。
> 更新日期：2026-04-04

---

## 更新目的

解决用户反馈的"发消息没反应"问题。

**原因**：之前主Agent直接执行法律任务，长任务期间用户收不到任何回复。

**解决方案**：主Agent变为调度员，法律任务委托给专业律师子Agent异步执行。用户发任务后立即收到确认，不再干等。

---

## 更新步骤

请按以下顺序执行。每一步完成后标记为完成。

### 第1步：创建律师子Agent

```bash
openclaw agents add lawyer
```

如果提示已存在，跳过此步。

### 第2步：创建律师Agent工作区

```bash
mkdir -p ~/.openclaw/workspace-lawyer
mkdir -p ~/.openclaw/agents/lawyer/agent
```

### 第3步：替换文件

将本目录下 `files/` 文件夹中的文件替换到对应位置：

| 源文件（本包） | 目标路径 | 说明 |
|---|---|---|
| `files/workspace-SOUL.md` | `~/.openclaw/workspace/SOUL.md` | 主Agent人设（调度员角色） |
| `files/workspace-AGENTS.md` | `~/.openclaw/workspace/AGENTS.md` | 主Agent配置（含委托协议） |
| `files/lawyer-AGENTS.md` | `~/.openclaw/workspace-lawyer/AGENTS.md` | 律师Agent领域规则 |
| `files/lawyer-SOUL.md` | `~/.openclaw/workspace-lawyer/SOUL.md` | 律师Agent人设说明 |
| `files/lawyer-agent.json` | `~/.openclaw/agents/lawyer/agent/agent.json` | 律师Agent配置 |

**替换命令示例**：

```bash
# 先备份原文件（重要！）
cp ~/.openclaw/workspace/SOUL.md ~/.openclaw/workspace/SOUL.md.bak
cp ~/.openclaw/workspace/AGENTS.md ~/.openclaw/workspace/AGENTS.md.bak

# 替换文件
cp files/workspace-SOUL.md ~/.openclaw/workspace/SOUL.md
cp files/workspace-AGENTS.md ~/.openclaw/workspace/AGENTS.md
cp files/lawyer-AGENTS.md ~/.openclaw/workspace-lawyer/AGENTS.md
cp files/lawyer-SOUL.md ~/.openclaw/workspace-lawyer/SOUL.md
cp files/lawyer-agent.json ~/.openclaw/agents/lawyer/agent/agent.json
```

### 第4步：修改 openclaw.json 配置

打开 `~/.openclaw/openclaw.json`，找到 `"agents"` 部分，将其替换为以下内容：

```json
"agents": {
  "defaults": {
    "workspace": "~/.openclaw/workspace"
  },
  "list": [
    {
      "id": "main",
      "workspace": "~/.openclaw/workspace",
      "subagents": {
        "allowAgents": ["lawyer"]
      }
    },
    {
      "id": "lawyer",
      "name": "lawyer",
      "workspace": "~/.openclaw/workspace-lawyer",
      "agentDir": "~/.openclaw/agents/lawyer/agent"
    }
  ]
}
```

**注意**：
- 只替换 `"agents"` 部分，其他配置（gateway、channels、plugins等）不要动
- `defaults` 中的 `workspace` 路径保持你原来的设置不变
- 关键是 `"list"` 数组中要有 `main` 和 `lawyer` 两个Agent，且 main 的 `subagents.allowAgents` 包含 `"lawyer"`

### 第5步：重启 Gateway

```bash
openclaw gateway restart
```

确认输出显示服务正常启动。

### 第6步：验证

在OpenClaw对话中发送一条法律任务，例如：

> "帮我分析一下盗窃罪的量刑标准"

**期望行为**：
1. 立即收到回复，如"收到，让律师处理一下。"
2. 几秒后收到律师的分析结果

**如果出现问题**：
- 回复"律师暂时不在" → 检查第4步 allowAgents 配置
- 回复"找不到律师" → 检查第1步和第3步文件位置
- 没有任何回复 → 检查第5步 Gateway 是否正常启动

---

## 回滚方案

如果更新后出现问题，恢复备份文件：

```bash
cp ~/.openclaw/workspace/SOUL.md.bak ~/.openclaw/workspace/SOUL.md
cp ~/.openclaw/workspace/AGENTS.md.bak ~/.openclaw/workspace/AGENTS.md
openclaw gateway restart
```
